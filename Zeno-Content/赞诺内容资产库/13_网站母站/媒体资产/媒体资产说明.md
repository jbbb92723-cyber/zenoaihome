# 媒体资产说明

> 本文档说明网站媒体资产的分类、命名规则和使用边界。  
> 最后更新：2026-04-19

---

## 一、目录说明

| 目录 | 放什么 | 是否进入网站 |
|---|---|---|
| `图片原图/` | 相机直出/手机原图，未压缩未裁剪 | ❌ 不进入网站，仅本地备份 |
| `视频原片/` | 录制的原始视频文件，体积大 | ❌ 不进入网站，上传至外部平台 |
| `封面图/` | 已压缩、已裁剪、已命名的封面图 | ✅ 可进入 `public/images/` |
| `文章配图/` | 文章正文内使用的配图，已压缩 | ✅ 可进入 `public/images/articles/` |
| `资料预览图/` | 资料库各资料的预览截图 | ✅ 可进入 `public/images/resources/` |
| `品牌头像/` | 个人头像、品牌 logo、签名图 | ✅ 可进入 `public/images/brand/` |

## 二、哪些图片可以进入网站

**可以进入网站的图片必须满足：**

1. 已使用 webp / jpg 格式压缩（原图不要直接放）
2. 文件大小：封面图 ≤ 200KB，配图 ≤ 150KB，头像 ≤ 80KB
3. 文件名已改为英文/拼音小写，无空格（见命名规则）
4. 图片尺寸已裁剪到合理比例（封面建议 16:9，头像建议 1:1）
5. 不包含个人隐私信息、证件或敏感内容

**不要进入网站的图片：**

- 相机原图（体积通常 3MB+，严重影响加载速度）
- 中文文件名的图片（URL 中文编码容易出错）
- 带水印或版权不明的图片
- 截图中含有身份证、合同、银行卡等敏感信息

## 三、图片命名规则

### 基本原则

- 全部使用小写英文或拼音
- 单词之间用连字符 `-` 分隔，不用空格、下划线或中文
- 文件扩展名使用 `.webp`（推荐）或 `.jpg`

### 文章配图按 slug 建文件夹

```
public/images/articles/
├── zhuangxiu-yusuan-weishenme-zongchao/
│   ├── cover.webp          ← 文章封面图
│   ├── scene-01.webp       ← 正文配图第1张
│   └── scene-02.webp       ← 正文配图第2张
├── jia-bu-shi-yangban-jian/
│   └── cover.webp
```

### 品牌图片

```
public/images/brand/
├── avatar.webp             ← 个人头像（200×200px）
├── avatar-square.webp      ← 方形头像（400×400px）
└── logo.svg                ← 品牌 logo（SVG 格式）
```

### 资料预览图

```

├── zhuangxiu-yusuan-moban-preview.webp
```

## 四、视频为什么不建议直接放进代码仓库

1. **体积问题**：一个 3 分钟视频通常在 50MB～300MB 之间。Git 仓库不适合存储大文件，会导致克隆缓慢、部署超时。

2. **带宽成本**：Vercel 等静态托管平台对流量计费，视频流量极高，免费额度会很快耗尽。

3. **播放体验差**：直接托管在 Vercel 的视频没有 CDN 加速，用户播放会卡顿。

4. **无法统计播放数据**：放到 YouTube / B站 / 视频号，天然有播放量、互动数据，有助于内容运营。

5. **维护成本高**：修改视频需要重新部署网站，而外部平台可以直接替换视频文件。

## 五、视频应该放到哪里

**推荐平台（按优先级）：**

| 平台 | 适合什么 | 说明 |
|---|---|---|
| **B站** | 中文长视频，社区互动 | 首选，中国用户体验最好 |
| **视频号** | 微信生态内传播 | 配合公众号使用 |
| **YouTube** | 国际传播，SEO | 境外流量和 AI 引用友好 |
| **腾讯云 COS** | 私有托管，嵌入播放 | 需要付费，适合不希望平台分流 |
| **Cloudflare R2** | 海外低成本对象存储 | 技术门槛略高，成本极低 |

**网站里只需要保存：**

```typescript
// data/videos.ts 里保存这些信息
{
  platform: 'bilibili',       // 平台名
  url: 'https://...',         // 视频主页链接
  embedUrl: 'https://...',    // 嵌入播放地址
  coverImage: '/images/videos/cover-01.webp'  // 封面图（本地）
}
```

## 六、压缩工具推荐

- **Squoosh**（在线，免费）：https://squoosh.app — 推荐用于压缩单张图片
- **TinyPNG**（在线，免费）：https://tinypng.com — 批量压缩 PNG/JPG
- **Photoshop / Lightroom**：导出时选择"为 Web 存储"，webp 格式
- **FFmpeg**（命令行）：处理视频封面截图

---

## 七、注意事项

- 每次往网站添加图片后，记得同步更新 `data/articles.ts`、`data/resources.ts` 等对应字段
- 图片加 `alt` 文案，对无障碍访问和 SEO 都有价值
- 原图永远保留在本目录，不要删除，方便日后重新处理
