| 表名 | 作用 | 第一阶段状态 |

## 一、profiles 表

**作用：** 存储用户基本信息和角色。是整个系统的用户主表。  
**为什么需要：** Auth.js 的 session 只携带基础信息，角色、昵称等扩展信息需要单独表存储。

```sql
CREATE TABLE profiles (
  id           TEXT PRIMARY KEY,          -- 来自 OAuth provider 的唯一 ID
  email        TEXT UNIQUE NOT NULL,
  display_name TEXT,
  avatar_url   TEXT,
  role         TEXT NOT NULL DEFAULT 'user',
                                          -- 取值：visitor | user | member | customer | admin
  created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_profiles_email ON profiles(email);
CREATE INDEX idx_profiles_role  ON profiles(role);
```

**role 取值说明：**

| role | 说明 |
|---|---|
| visitor | 未登录（前端判断，不存数据库）|
| user | 已注册登录，但未开通会员 |
| member | 免费会员 |
| customer | 付费会员 / 已购买用户 |
| admin | 管理员 |

**可以后续调整的字段：**  
- `bio`（个人简介）
- `wechat_openid`（微信 OpenID，接入微信登录时添加）
- `phone`（手机号，可选）
- `last_login_at`

**第一阶段状态：** OAuth 登录时自动写入，`role` 默认为 `user`。

## 二、memberships 表

**作用：** 管理用户的会员状态、等级和有效期。  
**为什么需要：** 一个用户可能有历史记录（过期会员→续费）。

```sql
CREATE TABLE memberships (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    TEXT NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  tier       TEXT NOT NULL DEFAULT 'free',
                                -- 取值：free | member | pro
  status     TEXT NOT NULL DEFAULT 'active',
                                -- 取值：active | expired | canceled
  started_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  expires_at TIMESTAMPTZ,       -- NULL 表示永久（如赠送会员）
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_memberships_user_id   ON memberships(user_id);
CREATE INDEX idx_memberships_status    ON memberships(status);
CREATE INDEX idx_memberships_expires   ON memberships(expires_at);
```

**tier 取值说明：**

| tier | 说明 |
|---|---|
| free | 免费会员（注册即获得）|
| member | 付费会员（基础）|
| pro | 高级会员 |

**status 取值说明：**

| status | 说明 |
|---|---|
| active | 有效 |
| expired | 已过期 |
| canceled | 主动取消 |

**可以后续调整的字段：**  
- `source`（来源：stripe / wechat / manual / gift）
- `order_id`（关联订单）

**第一阶段状态：** 建表，不启用真实会员购买逻辑。

## 三、resources 表

**作用：** 存储资料元数据、访问权限级别和文件链接。  
**为什么需要：** 资料权限需要持久化，不能只靠前端代码判断。

```sql
CREATE TABLE resources (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title         TEXT NOT NULL,
  slug          TEXT UNIQUE NOT NULL,     -- URL 友好标识
  description   TEXT,
  access_level  TEXT NOT NULL DEFAULT 'login',
                                          -- 取值：public | login | member | paid | admin
  file_url      TEXT,                     -- 真实文件链接（Supabase Storage 或 R2）
  preview_image TEXT,                     -- 预览图链接
  tag           TEXT,                     -- 分类标签
  created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_resources_slug         ON resources(slug);
CREATE INDEX idx_resources_access_level ON resources(access_level);
```

**access_level 取值说明：**

| access_level | 说明 |
|---|---|
| public | 所有人可查看和下载 |
| login | 登录后可领取 |
| member | 开通会员后可领取 |
| paid | 单独购买后可下载 |
| admin | 管理员专属，不对外显示 |

**第一阶段状态：** 建表备用，前端资料数据仍在 `data/resources.ts` 中维护。

## 四、resource_downloads 表

**作用：** 记录每次资料下载行为，用于防止滥用和统计热门资料。  
**为什么需要：** 权限验证后不能直接暴露真实文件链接，下载记录用于审计和限制。

```sql
CREATE TABLE resource_downloads (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       TEXT NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  resource_id   UUID NOT NULL REFERENCES resources(id) ON DELETE CASCADE,
  downloaded_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  ip_hash       TEXT          -- 对用户 IP 做哈希，不存明文 IP（隐私保护）
);

-- 索引
CREATE INDEX idx_downloads_user_id     ON resource_downloads(user_id);
CREATE INDEX idx_downloads_resource_id ON resource_downloads(resource_id);
```

**可以后续调整的字段：**  
- `download_count`（某用户对某资料的下载次数，用于限制多次下载）

**第一阶段状态：** 建表，不启用真实下载记录。

## 五、comments 表

**作用：** 存储用户评论，支持审核机制。  
**为什么需要：** 不开放匿名评论，必须登录。评论默认 pending，需管理员审核后才公开显示。

```sql
CREATE TABLE comments (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      TEXT NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  article_slug TEXT NOT NULL,   -- 对应文章的 slug（不用外键，灵活）
  content      TEXT NOT NULL,
  status       TEXT NOT NULL DEFAULT 'pending',
                                -- 取值：pending | approved | rejected | spam
  created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_comments_article_slug ON comments(article_slug);
CREATE INDEX idx_comments_user_id      ON comments(user_id);
CREATE INDEX idx_comments_status       ON comments(status);
```

**status 取值说明：**

| status | 说明 |
|---|---|
| pending | 待审核（默认）|
| approved | 已通过，公开显示 |
| rejected | 已拒绝 |
| spam | 垃圾信息，自动或手动标记 |

**可以后续调整的字段：**  
- `parent_id`（支持回复）
- `likes_count`（点赞数）
- `is_pinned`（置顶评论）

**第一阶段状态：** 建表，API 占位，评论提交后存入数据库待审核。

## 六、videos 表

**作用：** 存储视频元数据和访问权限。视频本身托管在外部平台，网站只保存链接和封面。  
**为什么需要：** 中心化管理视频的访问权限和关联内容。

```sql
CREATE TABLE videos (
  id                   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title                TEXT NOT NULL,
  slug                 TEXT UNIQUE NOT NULL,
  description          TEXT,
  platform             TEXT NOT NULL,     -- bilibili | youtube | weixin | tencent | other
  url                  TEXT NOT NULL,     -- 视频原始链接（点击跳转）
  embed_url            TEXT,             -- iframe 嵌入地址（可选）
  cover_image          TEXT,             -- 封面图路径
  access_level         TEXT NOT NULL DEFAULT 'public',
  related_article_slug TEXT,             -- 关联文章
  created_at           TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_videos_slug         ON videos(slug);
CREATE INDEX idx_videos_access_level ON videos(access_level);
```

**第一阶段状态：** 建表，前端数据仍在 `data/videos.ts` 中维护。

## 七、orders 表

**作用：** 记录用户的订单，包括付费资料、会员订阅等。  
**为什么需要：** 支付和权限必须解耦。订单是支付成功后开通权限的依据。

```sql
CREATE TABLE orders (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id           TEXT NOT NULL REFERENCES profiles(id) ON DELETE RESTRICT,
  product_type      TEXT NOT NULL,       -- membership | resource | service
  product_id        TEXT NOT NULL,       -- 对应 memberships.tier 或 resources.id 等
  amount            INTEGER NOT NULL,    -- 单位：分（人民币）/ 最小货币单位
  currency          TEXT NOT NULL DEFAULT 'CNY',
  status            TEXT NOT NULL DEFAULT 'pending',
                                         -- 取值：pending | paid | failed | refunded | canceled
  payment_provider  TEXT,               -- stripe | wechat | alipay | manual
  provider_order_id TEXT,               -- 第三方支付平台的订单 ID
  created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  paid_at           TIMESTAMPTZ          -- 实际支付时间
);

-- 索引
CREATE INDEX idx_orders_user_id  ON orders(user_id);
CREATE INDEX idx_orders_status   ON orders(status);
```

**status 取值说明：**

| status | 说明 |
|---|---|
| pending | 已创建，待支付 |
| paid | 已支付成功 |
| failed | 支付失败 |
| refunded | 已退款 |
| canceled | 已取消 |

**第一阶段状态：** 建表，API 路由占位，不接真实支付。

## 八、payment_events 表

**作用：** 记录支付平台的 webhook 回调原始数据。  
**为什么需要：** 支付回调必须做幂等处理（同一事件可能多次推送）。保留原始数据便于排查问题。

```sql
CREATE TABLE payment_events (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  provider    TEXT NOT NULL,   -- stripe | wechat | alipay
  event_type  TEXT NOT NULL,   -- payment.success | refund.completed 等
  raw_payload JSONB NOT NULL,  -- 支付平台推送的原始 JSON
  processed   BOOLEAN NOT NULL DEFAULT FALSE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_payment_events_provider   ON payment_events(provider);
CREATE INDEX idx_payment_events_processed  ON payment_events(processed);
```

**第一阶段状态：** 建表，不接收真实支付事件。

  avatarUrl   String?  @map("avatar_url")

  @@map("profiles")

## 初始化步骤（Supabase）

1. 在 [Supabase](https://supabase.com) 创建新项目
2. 进入 SQL Editor，按顺序执行上面的 CREATE TABLE 语句
3. 在项目 Settings → Database → Connection string 获取 `DATABASE_URL`
4. 将 `DATABASE_URL` 填入 `.env.local`
