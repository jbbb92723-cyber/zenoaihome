# 内容工厂 SOP

> 这不是文档。这是你的操作清单。
> 每次打开 Obsidian，按这张表走。不用思考"该做什么"。

---

## 每天（5分钟）

- [ ] 打开 `01原始素材/灵感碎片/`，今天有什么新想法？随手记一条。
- [ ] 有没有业主问了新问题？扔进 `01原始素材/业主提问/`。

## 写文章时（按流程走）

```
① 选题从哪来？
   → 01原始素材/业主提问/ — 找真实问题
   → 02内容单元库/ — 找已有概念拼装
   → 06选题装配/ — 找已有草稿

② 怎么写？
   → 06选题装配/ → 新建草稿.md
   → Ctrl+O 搜 [[概念名]] → 用 wikilink 引用
   → 写完，确认 CTA 在结尾

③ 怎么发？
   → 草稿 → 告诉 Hermes "把这篇文章部署到网站"
   → Hermes 生成 article-*-content.ts → next build → 上线
```

## 发布后24小时内（15分钟）

- [ ] 打开刚发布的文章，拆出 2-3 个内容单元：
  - 有新概念？→ `02内容单元库/CON-概念单元/`
  - 有案例？→ `02内容单元库/CAS-案例单元/`
  - 有可执行方案？→ `02内容单元库/SOL-方案单元/`
- [ ] 在 `03处理状态/处理日志.md` 记一行
- [ ] 在 Obsidian 草稿 frontmatter 加 `published_as: article-xxx`

## 每周（15分钟）

- [ ] 扫一眼 `03处理状态/处理日志.md` — 哪篇还没拆零件？
- [ ] 清理 `01原始素材/` — 哪些灵感过时了？删。
- [ ] 在 Obsidian Graph View 看连接密度 — 连接越密 = 知识越体系化

## 关键原则（don哥的5条）

1. **发布即拆**：文章上线24小时内拆零件。超过24小时，大概率永远不拆。
2. **不追求完美**：内容单元只需要一句话定义+来源链接。用到时再补细节。
3. **先跑通闭环**：拆2篇文章的零件→复用一次→闭环验证。不要等到"系统完美"才开始。
4. **一个文件一个来源**：每张内容单元卡标注来源。可追溯。
5. **文件夹不膨胀**：不再加新顶级目录。如果有新需求，先看看能不能放进现有的8个里。
