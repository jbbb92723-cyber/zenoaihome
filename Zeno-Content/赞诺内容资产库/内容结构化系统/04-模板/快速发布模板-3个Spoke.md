# 3个Spoke快速模板

> Justin Welsh: "不是写得更快，是少写新东西。有模板，拆Spoke从30分钟→5分钟。"

---

## 模板1：Anti-Vision帖（小红书/公众号）

**结构**：[具体数字的痛苦场景] → [为什么发生] → [不解决的代价] → [出口]

```
【标题：一个不[X]的业主，3个月后发生了什么】

[真实业主场景，具体到数字]

她装修预算[数字]，签合同之前[做了什么/没做什么]。3个月后，她花了[数字]。

不是因为对方骗了她。是[核心原因——信息差/一句话没写清楚]。

---

如果你正在装修，这种事每天都在发生。区别只有一个：签合同之前，有没有人帮你看一眼报价单。

→ 把报价单拍照发我，24小时内出结果。¥199。审不出问题，退款。
```

**填写指南**：
- [数字] 必须真实或可信——"15万变23万"比"大量超支"有力100倍
- 不制造焦虑。结尾是出口，不是恐惧。

## 模板2：Before/After帖（小红书图文）

**结构**：[Before场景] vs [After场景] → [做了同一件事的不同结果]

```
【同一个小区的两个业主，同样的房子，同样的装修公司】

业主A：[做了什么] → 结果：[不好的结果+数字]

业主B：[做了同一件事但多了一个步骤] → 结果：[好的结果+数字]

他们之间的区别不是运气。是[核心动作]。

---

你不需要变成装修专家。你只需要在做[关键决策]之前，有一个人帮你看一眼。

→ 把报价单发给我。¥199，24小时。
```

**填写指南**：
- Before和After必须指向同一个动作的"做"和"没做"
- 数字对比必须有冲击力

---

## 模板3：Checklist帖（小红书图文/公众号）

**结构**：[一个常见的错误认知] → [3个自查问题] → [出口]

```
【你以为[常见误判]，其实[真相]】

签合同之前，先问自己3个问题：

1. [问题1——检查模糊表述]
2. [问题2——检查隐藏加价]
3. [问题3——检查责任边界]

如果3个问题你都回答不出来——把报价单发给我。我帮你回答。

→ ¥199，24小时。审不出问题，退款。
```

**填写指南**：
- 3个问题要具体、可执行——不是"你了解报价吗"，是"你圈出所有'暂估''另计''不含'的词了吗"
- 结尾的"发给我"是唯一CTA。不要放多个链接。
