# OA后台改建方案：从数据仓库到运营中控台

## 当前问题

现有结构按数据类型分组（用户/内容/营销/系统），无法回答三个核心问题：
1. 这个月赚了多少钱？从哪个产品赚的？
2. 一个业主从"读到文章"到"付费¥2,500"走了几步？哪步断了？
3. 现在有几个客户在服务中？交付进度到哪了？

## 改建原则

- 一人公司不需要复杂权限系统，但需要**一眼看到钱和客户状态**
- 每个数字大屏只回答一个问题，不堆砌
- 先做能直接驱动收入决策的模块，其他按需加

---

## 新结构：六大中控面板

### ① 收入总览（首页大屏）

**唯一目的**：看一眼就知道这个月赚了多少。

| 卡片 | 内容 |
|------|------|
| 本月收入 | ¥XX,XXX（实时汇总所有付费订单） |
| 本月订单数 | X 笔 |
| 收入拆解 | 饼图：¥2,500审查 / ¥499快审 / ¥199初查 / 节点顾问 / 床垫 |
| 客单价趋势 | 近6个月折线 |
| 待处理订单 | X 笔（已付款未交付） |

**MVP优先级：P0。没有这个，其他都是废的。**

---

### ② 转化漏斗（运营中控）

**唯一目的**：看清流量从哪来、卡在哪。

```
文章阅读 → 免费工具使用 → 微信添加 → 咨询 → 付费
   ↓           ↓              ↓        ↓       ↓
  UV数      完成人数      添加数    咨询数   成交数
```

每个节点显示转化率。比如：
- 免费初筛 100人完成 → 5人加微信 = 5%
- 5人加微信 → 2人咨询 = 40%
- 2人咨询 → 1人付费 = 50%

**你需要知道的是：哪个转化率最低，那个环节就是你该优化的。**

| 子模块 | 说明 |
|--------|------|
| 流量来源 | 各渠道UV（搜索引擎/直接访问/社交媒体） |
| 工具使用 | 免费初筛完成数、居住诊断完成数 |
| 微信转化 | 手动标记：哪天加的、备注了什么（初期不做自动对接） |
| 付费转化 | 从"加微信"到"下单"的天数和转化率 |

**MVP优先级：P0（核心指标手动录入即可，不要做自动对接）**

---

### ③ 客户中心

**唯一目的**：知道每个客户在哪个阶段。

| 列表字段 | 说明 |
|----------|------|
| 姓名/昵称 | |
| 微信 | |
| 来源 | 免费工具/公众号/朋友圈/转介绍 |
| 当前阶段 | 浏览中 / 已加微信 / 已咨询 / 已下单 / 服务中 / 已完成 |
| 最近服务 | ¥2,500审查 / 节点顾问 等 |
| 最后联系 | 日期 |
| 备注 | 自由文本 |

点击客户进入详情：完整动线时间线 + 所有订单记录 + 沟通备注。

**MVP优先级：P0。一人公司不需要CRM，但要一个能搜、能记的客户通讯录。**

---

### ④ 服务交付台

**唯一目的**：别让已付款的客户等着。

| 列表字段 | 说明 |
|----------|------|
| 客户名 | |
| 服务类型 | ¥2,500审查 / 节点顾问 |
| 状态 | 待交付 / 交付中 / 已交付待确认 / 已完成 |
| 承诺时间 | 审查24h/快审2-3天 |
| 剩余时间 | 倒计时 |

逾期高亮标红。一人公司最容易出问题的不是"卖不出去"，是"卖了忘了交付"。

**MVP优先级：P0。**

---

### ⑤ 内容工场（已有基础，补齐即可）

现有：内容草稿、判断笔记、居住判断。这些保留。

需要加的：

| 子模块 | 说明 |
|--------|------|
| 发布日历 | 哪些文章已发、哪些在排期 |
| CTA追踪 | 每篇文章底部的CTA点击量（免费工具/服务页） |
| 文章表现 | 每篇UV、停留时间、转化到工具的比率 |

**MVP优先级：P1。内容工场已有基础，CTA追踪是增量。**

---

### ⑥ 系统日志（已有，保留）

操作日志、行为事件——保留，不需要改。

## 生财有术的施工建议

1. **MVP思维**：先做收入总览。不画饼图不画折线——就一个数字：本月收入。这个数字对了，其他可以慢慢加。
2. **第一块钱思维**：转化漏斗初期手动更新。每周花10分钟填几个数字——不要等系统对接好再看，那可能永远等不到。
3. **算账思维**：收入总览 + 服务交付台 = 运营中控的MVP。这两块能告诉你"赚多少"和"欠谁交付"。其他都是锦上添花。
4. **迭代思维**：不要在后台建设上花超过一周。能用就行，不好用改。完美后台是改出来的不是设计出来的。
