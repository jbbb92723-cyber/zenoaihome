# 内容复用日历

## 单篇深度文章的标准发布流水线

```
Week 1 — Hub发布周
├── Day 1: 公众号发布总纲（2000-3500字深度文章）
├── Day 2: 小红书Spoke 1-2（痛点/故事切入）
├── Day 3: 小红书Spoke 3-4（数字/清单切入）
├── Day 4: 短视频（60秒口播）
├── Day 5: 轮播图发布 + 推文/朋友圈

Week 3 — 二次分发
├── Day 17: Spoke 3换标题重发（覆盖新受众）
├── Day 19: 短视频换开头重发

Week 9 — 三次分发
├── Day 60: 总纲换开头重发（"3个月后回头看"角度）
└── Day 62: Spoke 1换标题重发
```

## 规则

1. **不发Spoke不发主篇**：写完Hub后，至少拆出3条Spoke，主篇和Spoke同天或次天发
2. **内容复用 > 新创作**：每周新创作的Hub不超过2篇。其他时间用来复用旧内容
3. **CTA必查**：每次发布前运行 `python Zeno-Tools/scripts/publish.py` 检查CTA
4. **数据记录**：每次发布后记录发布平台、时间、互动数据，为下次复用提供依据

## 选题装配到发布的状态机

```
选题构思 → 内容单元调用 → 装配稿 → 生产单 → Spoke拆解 → CTA检查 → 发布 → 数据复盘 → 加入复用队列
                                                                                    ↓
                                                                              3轮复用
```
