# 命名规则

## 内容单元命名

**格式**：`{类型码}-{YYYYMMDD}-{编号}_{名称}.md`

**示例**：
- `CON-20260710-001_报价单=责任边界表.md`
- `CAS-20260710-001_15万变23万的业主.md`
- `OPI-20260710-001_签约前谈规则成本最低.md`
- `SOL-20260710-001_签约前自检清单.md`
- `QUE-20260710-001_按实结算规则够不够.md`

**编号规则**：同类型下按创建顺序递增（001, 002, 003...），跨天不重置。

## 类型码

| 码 | 含义 |
|----|------|
| CON | 概念单元 |
| CAS | 案例单元 |
| OPI | 观点单元 |
| SOL | 方案单元 |
| QUE | 问题单元 |

## 其他文件命名

| 目录 | 格式 | 示例 |
|------|------|------|
| 选题装配 | `YYYY-MM-DD_标题.md` | 已有格式，不变 |
| 模板 | `TPL-YYYYMMDD-编号_模板名.md` | 已有格式，不变 |
| 系统复盘 | `RETRO-YYYYMMDD_标题.md` | 已有格式，不变 |
| 产品服务 | `PRD-YYYYMMDD-编号_名称.md` | 已有格式，不变 |
| 系统配置 | `SYS-YYYYMMDD-编号_名称.md` | 已有格式，不变 |

## Obsidian 链接规则

- 引用其他内容单元时：`[[CON-20260710-001_报价单=责任边界表]]`
- 引用选题装配稿时：`[[2026-07-09_报价单责任边界]]`
- 不要用完整路径，用 `[[文件名]]`
