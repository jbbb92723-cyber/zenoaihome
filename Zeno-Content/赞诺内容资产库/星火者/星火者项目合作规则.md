# 星火者 · 项目合作规则

> 这份文档规定共燃项目的协作规则。每个项目启动前，所有参与者确认本规则。

---

## 一、项目类型

| 类型 | 代号 | 频率 | 时长 | 人数 |
|------|------|------|------|------|
| 月度探索 | **共燃·猎场** | 视发起情况 | 30天 | 3-8人 |
| 季度深度 | **共燃·炉边** | 每季1次 🔵计划中 | 2天线下 | 10-20人 |
| 年度大型 | **共燃·燎原** | 每年1次 🔵计划中 | 3个月 | 不限 |

> 首期试运行只启动猎场。炉边和燎原为计划中，待核心成员数量达到阈值后启动。

---

## 二、发起规则

### 2.1 发起条件

- 发起人（点火人）需为核心星火者或主理人特批
- 在成员主群发起

### 2.2 发起必须说清五件事

1. **做什么**：项目目标（一句话能说清）
2. **需要谁**：角色 + 技能要求（至少列出2个角色）
3. **分润方案**：收益分配比例（必须在组队前公布）
4. **时间线**：启动、里程碑、结项日期
5. **交付标准**：什么算成功，什么算失败

### 2.3 发起模板

```
🔥 猎场招募：第00X号

【项目名称】一句话说清楚
【点火人】[名字]
【一句话目标】30天内要拿到什么结果
【项目描述】3-5句话说明白要做什么
【所需角色】
  · [角色A]：[技能要求] | 1人
  · [角色B]：[技能要求] | 1-2人
【分润方案】总收益分配比例（组队前公布，不可后改）
【时间线】
  启动：X月X日
  Kickoff会议：X月X日（组队后48h内）
  结项交付：X月X日
【报名方式】评论区报名 + 一句话介绍相关经验
【截止时间】X月X日
```

## 三、合同与主体

### 3.1 签约主体

涉及企业客户的项目，由**广西赞诺数智科技有限公司**或指定项目主体与客户单独签署服务合同。

### 3.2 成员参与

成员参与交付时，另行签署项目合作确认文件，明确以下内容：

| 事项 | 必须写清 |
|------|----------|
| 客户是谁 | 甲方全称 |
| 谁签合同 | 签约主体 |
| 谁收款和开票 | 收款账户、开票方 |
| 项目总价 | 合同总金额 |
| 成本如何计算 | 哪些算成本、谁承担 |
| 收益分配 | 收益还是净利润参与分配、具体比例 |
| 分工与交付 | 每人负责什么、验收标准 |
| 时间节点 | 每个节点的交付物和截止日期 |
| 延期处理 | 延期如何通知、如何处理 |
| 退款和违约 | 客户退款如何处理、成员违约如何处理 |
| 知识产权 | 交付物归属、成员可否复用 |
| 保密义务 | 客户信息的保密范围和期限 |
| 平台是否承担交付责任 | 明确！ |

### 3.3 平台管理费

> 涉及企业客户的项目，具体服务费、成本、结算方式和平台管理费，以项目合同为准，不以本手册中的比例作为自动承诺。

## 四、参与规则

### 4.1 报名与组队

- 自由报名，点火人拥有最终选择权
- 组队后48小时内，在临时项目群开kickoff会议
- 团队人数上限8人

### 4.2 中途退出

- 不扣星火值
- 需及时通知点火人和团队
- 退出者的未完成工作由团队重新分配
- 退出记录影响社区信用

### 4.3 进度跟踪

- 每周在项目群发布进度简报（做了什么/卡在哪/下周计划）
- 连续2周无进度更新 → 主理人介入

## 五、结项与复盘

### 5.1 结项标准

| 交付物 | 成功项目 | 失败项目（客观原因） | 失败项目（主观跳票） |
|--------|----------|---------------------|---------------------|
| 项目复盘 | ✅ | ✅ | — |
| 数据/结果 | ✅ | — | — |
| 失败分析 | — | ✅ | — |
| 星火值 | +20 | +10 | -15 |

### 5.2 结项模板

```
🚀 猎场#00X 结项复盘

【项目名称】
【点火人】[名字]
【共燃者】[列出所有参与者]
【时间周期】X月X日 - X月X日（共X天）

【项目结果】
  · 目标：[初始目标]
  · 实际：[实际结果，有数据就用数据]
  · 差距分析：[如果没达成，为什么]

【过程回顾】
  第1周：[做了什么 / 卡在哪里]
  第2周：...
  第3周：...
  第4周：...

【关键收获】
  1. [学到的最重要的事]
  2. [意外的发现]
  3. [如果再做一次会怎么优化]

【踩过的坑】
  坑1：[什么坑 / 怎么掉进去的 / 怎么爬出来的]

【团队评价】（每个成员写1-2句话）

【是否可以开源】
  是 / 部分可以 / 否
```

### 5.3 结项后

- 项目群归档 → 解散
- 交付物上传知识星球猎场档案区
- 复盘内容经脱敏后发布到知识星球

## 六、知识产权

- 项目交付物的知识产权归属以项目合作确认文件为准
- 如无特别约定，交付物著作权归客户（甲方）
- 成员个人在项目中贡献的方法、流程、经验，成员个人保留复用权
- 项目复盘内容经脱敏后，星火者共同体可在内部使用

---

## 七、争议处理

### 7.1 一般争议

成员间争议 → 先由当事双方直接沟通 → 无法解决的，主理人介入调解。

### 7.2 与客户争议

由签约主体（广西赞诺数智科技有限公司或指定项目主体）与客户按合同约定处理。成员配合提供相关交付记录。

### 7.3 分润争议

- 分润方案在组队前公布，组队后不可更改
- 如有争议，以组队前公布的分润方案为准
- 实际分配以客户实际付款金额为基础

---

## 八、风险提示与免责

- 共燃项目是成员自愿发起的协作，不构成雇佣关系或合伙关系
- 项目收益取决于市场、客户、执行等多种因素，星火者平台不保证项目成功或盈利
- 成员参与项目应自行评估风险和自身能力
- 平台不对项目结果承担交付责任或连带责任（以项目合同和项目合作确认文件为准）
