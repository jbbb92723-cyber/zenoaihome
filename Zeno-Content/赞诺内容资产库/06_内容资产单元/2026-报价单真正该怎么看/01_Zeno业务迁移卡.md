## 1. 这条机制如何服务于报价风险判断

ZenoAIHome 首页写的是：

> "今天在看的：南宁第 38 份水电没有上限的报价——和 2018 年第一份一样。"

这句话本身就是"报价单是责任边界表"的现场版表达。

具体来说，这个机制服务于以下判断：

### 报价初筛（免费工具）
业主上传报价单 → 系统检查的不是价格高低，而是**边界是否清楚**：
- 有没有项目遗漏？
- 有没有"另计""暂估""不含""按实结算"？
- 材料有没有写型号？
- 水电有没有上限？

→ 输出不是"这个报价贵不贵"，而是"这份报价的边界你还没搞清楚的地方"。

### 报价初查（¥199）
人工复核 10 行以内的重点报价 → 做的事是：
- 把模糊词翻译成风险
- 标出 3 个高风险点
- 给出 5 个签约前追问问题

→ 这不是"告诉你选哪家"，而是"告诉你签约前必须问清楚什么"。

### 报价/合同快审（¥499）
完整报价 + 合同草稿 + 付款节点放在一起看 → 做的事是：
- 漏项/模糊项/增项口子检查
- 付款节点风险提醒
- 建议写入备注的内容

→ 核心判断逻辑完全来自"报价单是责任边界表"这个底层机制。

### 居住方案综合判断（¥1999）
把生活目标、预算、报价、合同、交付风险放在一起看 → 这件事的起点是：
- 报价单覆盖的范围，跟你想要的生活，对得上吗？

## 2. 这个机制在网站上的表达位置

### 首页
- "从哪开始"三个入口 → 报价风险初筛的定位就是"先判断边界，再比价格"
- "签约前必看"三个风险（R1按实结算 / R2材料只写品牌 / R3口头承诺）→ 每一项都是"边界没说清"

### 博客
- 报价单真正该怎么看 → 本文，核心转化文章
- 装修预算为什么总超 → 姊妹篇，讲"预算边界"
- 14篇报价风险系列 → 每个单篇拆一个具体边界问题

### 工具/词典
- 报价审核清单 → 六大板块逐项检查清单，本质是"边界检查表"
- 风险词典 → 把"另计""暂估""不含""按实结算"等模糊词翻译成可检查的问题

## 3. 转化路径中的角色

```text
业主搜"装修报价单怎么看"
→ 搜到这篇文章
→ 文章不讲"怎么比价"，讲"怎么看边界"
→ 文章底部引导到报价初筛工具（免费）
→ 初筛暴露更多边界问题
→ 引导到报价初查 / 快审（付费）
→ 业主在签约前把边界搞清楚
→ 签约后的增项风险大幅降低
```

这篇文章在漏斗里的角色不是"信息文章"——它是**报价审核服务的信任入口**。

它要完成的事是：

> 让读者看完之后，不再觉得"比价"是看报价的正确方法，而是意识到"边界清楚"才是判断标准。

## 4. 服务边界说明

这篇文章和报价审核服务的关系：

**文章负责：** 建立判断框架——报价单不是比价工具，是责任边界表。

**服务负责：** 替业主执行这个框架——逐项检查、标出风险、给出追问清单。

**文章不做：** 替业主判断哪家好、推荐施工方、代替律师审合同。

**服务不做：** 只看总价就说"这个便宜"——因为边界不清的情况下，便宜没有意义。
