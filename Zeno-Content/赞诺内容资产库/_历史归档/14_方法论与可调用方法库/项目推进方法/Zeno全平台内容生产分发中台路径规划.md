# Zeno 全平台内容生产分发中台路径规划

> 目标：把 Hermes 生成的内容、图片、资料包，不再困在本地电脑，而是进入一个可在手机端审核、保存、发布、复盘的“内容分发中台”。

## 一句话架构

```text
Obsidian/Zeno OS = 资产真源
Hermes = 生产与调度引擎
云盘 = 手机可访问的媒体交付层
飞书 = 审批/状态/任务中控台
平台端 = 小红书 / 公众号草稿箱 / 闲鱼 / 网站 / 私域
复盘数据 = 回填到飞书 + Obsidian
```

## 1. 为什么需要云盘层

当前痛点：

- 图片和文案生成在本地电脑。
- Zeno 不一定在电脑旁。
- 小红书、闲鱼等平台主要依赖手机端发布。
- 公众号需要进入草稿箱。
- 虚拟资料需要可交付、可下载、可追踪。

云盘层解决的是：

```text
本地生产出来的文件，如何自动变成手机可拿、飞书可看、平台可发的发布资产。
```

它不是知识库真源，真源仍然是 Obsidian/Zeno OS；云盘只是“发布交付层”。

## 2. 推荐总架构

```text
外部信号
  ↓
内容雷达：热点搜索 / 对标博主 / 评论区痛点 / 私信咨询
  ↓
Hermes 生产：标题 / 文案 / 图片 / 标签 / CTA / 资料包
  ↓
Obsidian 入库：方法卡 / 痛点卡 / 内容资产单元 / SOP
  ↓
云盘同步：图片包 / 发布包 / 虚拟资料包
  ↓
飞书中控：状态、审核、发布计划、链接、复盘
  ↓
分平台发布：
  - 小红书：手机手动/半自动发布
  - 公众号：API/浏览器保存草稿箱
  - 闲鱼：半自动发布/商品资料维护
  - 网站：Next.js/VSCode 发布流
  ↓
复盘回填：数据、评论、咨询、成交线索
```

## 3. 三层文件系统

### A. Obsidian/Zeno OS：资产真源

保存长期可复用资产：

```text
G:/Zenoaihome.com/Zeno-Content/赞诺内容资产库/
├── 06_内容资产单元/
├── 16_提示词库/
├── 14_方法论与可调用方法库/
├── 22_运行记录与复盘/
```

原则：

- 长期方法、拆解、复盘、内容包真源都在这里。
- 不把云盘当知识库。
- 不在云盘里做复杂版本管理。

### B. 云盘：发布交付层

建议建立一个单独同步目录，例如：

```text
Zeno发布中台/
├── 00_INBOX_待处理/
├── 10_XHS_小红书/
│   ├── 待审核/
│   ├── 待发布/
│   ├── 已发布/
│   └── 素材库/
├── 20_WECHAT_OA_公众号/
│   ├── 待成稿/
│   ├── 待入草稿箱/
│   ├── 已入草稿箱/
│   └── 封面图/
├── 30_XIANYU_闲鱼/
│   ├── 商品资料/
│   ├── 待发布/
│   ├── 已发布/
│   └── 客服话术/
├── 40_VIRTUAL_PRODUCTS_虚拟资料/
│   ├── 交付包/
│   ├── 封面图/
│   ├── 预览图/
│   └── 购买后说明/
├── 50_WEBSITE_网站/
│   ├── 待更新/
│   └── 发布素材/
└── 90_ARCHIVE_归档/
```

每个内容包统一结构：

```text
YYYY-MM-DD-{platform}-{topic}/
├── 00_publish.md        # 发布文案，可复制
├── 01_images/           # 平台图片
├── 02_cover/            # 封面
├── 03_assets/           # 附件/资料包
├── 04_review.md         # 审核记录
└── 05_result.md         # 发布链接和数据复盘
```

### C. 飞书：状态中控层

飞书不替代 Obsidian，也不替代云盘。它负责：

- 今天有什么待审核。
- 哪些已生成。
- 哪些已入草稿箱。
- 哪些已发布。
- 哪些要复盘。
- 哪些需要 Zeno 判断。

## 5. 分平台自动化路径

### A. 小红书

推荐：云盘同步 + 手机发布。

流程：

```text
Hermes 生成图片/文案
→ 写入 Obsidian 内容包
→ 复制到云盘 10_XHS_小红书/待发布/
→ 飞书状态改为「待发布」
→ Zeno 手机打开云盘保存图片
→ 小红书 App 发布
→ 发布链接发回 Hermes/飞书
→ 复盘回填
```

原因：

- 小红书无稳定公开发布 API。
- 自动化发布账号风险较高。
- 手机端发布最稳定。

目标不是全自动发布，而是：

```text
手机 2 分钟发布。
```

### B. 公众号

推荐：优先打通草稿箱。

可用工具：`baoyu-post-to-wechat`。

两种方式：

1. API 入草稿箱：最快，但需要公众号 API 凭证和 IP 白名单。
2. 浏览器入草稿箱：较慢，但适合没有 API 条件时使用。

流程：

```text
Obsidian/Markdown 正文
→ baoyu-post-to-wechat 转换主题 HTML
→ 上传封面和正文图片
→ 保存到公众号草稿箱
→ 飞书状态：已入草稿箱
→ Zeno 手机/电脑最终审核发布
```

建议优先级：高。公众号草稿箱比小红书更适合自动化。

### C. 闲鱼

推荐：先做“商品资料包自动生成”，暂不急着自动发布。

原因：

- 闲鱼也偏移动端，自动发布容易遇到风控。
- 真正重要的是商品标题、主图、详情、交付文件、客服话术。

流程：

```text
虚拟资料主题
→ 生成资料 PDF/表格/清单
→ 生成商品主图/详情图
→ 生成闲鱼标题/文案/客服话术
→ 云盘 30_XIANYU_闲鱼/待发布/
→ 手机端手动发布或后续接浏览器自动化
```

可卖资料方向：

- 装修报价风险检查表
- 签合同前 30 个追问清单
- 半包报价核对表
- 增项风险词典
- 材料品牌型号确认表

### D. 虚拟资料交付

推荐：云盘 + 飞书记录 + 微信/闲鱼客服话术。

流程：

```text
资料生成
→ PDF/表格/图片包
→ 云盘 40_VIRTUAL_PRODUCTS_虚拟资料/交付包/
→ 生成短链接/提取说明
→ 飞书商品表记录
→ 闲鱼/微信成交后发送交付说明
```

注意：

- 免费预览版和付费完整版分开。
- 每个资料包要有版本号。
- 交付链接不要直接暴露 Obsidian 真源目录。

### E. 网站

继续保持：Obsidian/内容资产 → zeno-site 数据文件 → VSCode/部署。

不建议让所有小红书内容都进网站。只有经过验证的高价值选题，才沉淀成：

- FAQ
- 风险词条
- 服务页补充
- SEO/GEO 文章

## 6. 自动化等级设计

### L0：本地生成

Hermes 只生成文件，存在本地。

### L1：云盘同步

Hermes 自动把发布包复制到云盘同步目录。手机可访问。

### L2：飞书中控

Hermes 同步创建/更新飞书任务，带云盘链接、状态、审核字段。

### L3：平台草稿

公众号等支持的渠道自动入草稿箱。小红书/闲鱼仍保留手机确认。

### L4：自动发布

仅对风险低、API 稳定、可撤回的平台开放。小红书/闲鱼暂不建议进入 L4。

## 7. 推荐落地顺序

### 第 1 周：打通云盘发布包

目标：任何内容生成后，都能自动进入手机可访问目录。

动作：

1. 选择云盘：百度网盘 / 夸克 / OneDrive / Google Drive / iCloud / 坚果云。
2. 在本机建立同步根目录。
3. 固定发布包目录结构。
4. Hermes 生成内容后自动复制图片和 `publish.md` 到云盘。
5. 微信/飞书推送云盘路径和发布摘要。

验收：手机能在 1 分钟内找到并保存图片。

### 第 2 周：打通飞书中控

目标：所有待发布任务有状态。

动作：

1. 建飞书内容任务表。
2. 字段先少：标题、平台、状态、云盘链接、Obsidian 路径、审核意见、发布链接、复盘。
3. Hermes 每次生成内容包时更新一条任务。
4. Zeno 手机端改状态。

验收：每天知道哪些内容待发、已发、待复盘。

### 第 3 周：公众号入草稿箱

目标：公众号文章自动进入草稿箱。

动作：

1. 配置 `baoyu-post-to-wechat`。
2. 优先尝试 API；不行用浏览器法。
3. 固定公众号 Markdown 模板。
4. 生成后自动入草稿箱，不自动群发。

验收：公众号后台能看到草稿。

### 第 4 周：闲鱼虚拟资料包

目标：先把商品资料自动化，而不是急着自动发商品。

动作：

1. 做 1 个低价虚拟资料 MVP。
2. 自动生成商品图、标题、详情、客服话术。
3. 云盘准备交付包。
4. 手机端手动发布闲鱼。
5. 成交/咨询反馈回填。

验收：能形成从内容痛点 → 资料包 → 闲鱼商品 → 咨询反馈的闭环。

## 8. 每日运行样例

```text
08:00  Hermes 抓取/接收素材，生成内容雷达
08:30  生成今日小红书内容包和图片
08:40  写入 Obsidian
08:41  同步到云盘 /10_XHS_小红书/待发布/
08:42  飞书创建任务：待发布
08:43  微信推送：今日待发布摘要 + 云盘路径
中午   Zeno 手机发布小红书
晚上   Zeno 把链接发回 Hermes
次日   Hermes 记录数据和评论痛点
```

## 9. 风险控制

### 必须人工确认的平台

- 小红书
- 闲鱼
- 任何涉及交易、私信、自动回复、价格承诺的平台

### 可以自动入草稿的平台

- 公众号草稿箱
- 飞书文档
- Obsidian 内容包
- 云盘交付包

### 可以自动发送的平台

- 微信/飞书给 Zeno 自己的提醒
- 内部日报
- 已确认的资料交付说明（后续再做）

## 10. 最小技术实现

本地配置一个环境变量：

```text
ZENO_PUBLISH_SYNC_DIR=云盘同步根目录/Zeno发布中台
```

Hermes 每次生成内容包后执行：

```text
1. 在 Obsidian 写入内容资产单元。
2. 在云盘目录创建平台发布包。
3. 复制图片、PDF、封面、publish.md。
4. 发送微信/飞书摘要。
5. 后续接入飞书表 API 更新任务状态。
```

## 11. 验收标准

第一阶段成功标准：

- 手机能访问当天小红书图片和文案。
- 公众号文章能进入草稿箱或至少形成草稿包。
- 闲鱼商品能形成标题、主图、详情、交付包。
- 每条内容都有 Obsidian 真源路径和云盘发布路径。
- 发布后至少能回填链接和关键数据。

## 一句话提醒

**不要把所有平台都做成全自动发布；先把“生产、同步、审核、草稿、手机发布、复盘”打通。自动化的核心不是替你点发布，而是让你随时随地能安全地发。**
