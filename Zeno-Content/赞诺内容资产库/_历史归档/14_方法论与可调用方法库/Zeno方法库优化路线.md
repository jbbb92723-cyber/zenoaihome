# Zeno 方法库优化路线

> 目标：把现有方法库从“方法集合”升级成“可调用、可执行、可转化、可复盘”的判断资产系统。

## 当前判断

方法库已经有骨架，问题不是缺方法，而是缺少统一总纲、执行字段和服务转化挂钩。

当前已有优势：

- 已有方法分类：内容表达、商业判断、项目推进、流量选题。
- 已有核心方法：一问三用、语言校准、素材膨胀、需求倒推、内容资产闭环。
- 已有项目落点：ZenoAIHome 与 OPC 一人公司 AI 培训。
- 已有 Agent 执行层：Hermes Skill、dbskill、gstack、Obsidian 工作流。

当前主要缺口：

- 方法卡格式不完全统一。
- 有些方法偏“读得懂”，但还不能直接执行。
- 方法与服务 SOP、报价审核字段、内容转化路径的连接还不够硬。
- 复盘回填还没有成为每个方法的默认闭环。

## 优化后的方法卡标准

每张高价值方法卡建议补齐 8 个字段：

```markdown
# 方法卡标题

## 一句话用途
这张方法解决什么问题。

## 适用场景
什么时候调用它。

## 输入
需要什么材料：截图、链接、客户问题、报价单、旧观点、文章等。

## 操作步骤
1. ...
2. ...
3. ...

## 输出
最后应该产出什么：判断、清单、内容素材包、服务脚本、方法卡、Skill。

## 质量标准
什么样算合格，什么样算浅。

## 可转化资产
- 内容资产：
- 服务资产：
- 系统资产：

## 复盘回填
执行后要把什么数据或反馈写回哪里。
```

## 方法库四层分类

### 1. 方法卡

给人读，帮助 Zeno 快速回忆判断。

标准：短、清楚、可调用，不写成长篇理论。

### 2. 操作法

给 Agent 执行。

标准：有输入、步骤、输出模板、质量门槛。

候选升级对象：

- 爆款选题双轨素材膨胀法
- 内容资产转化纠偏
- 一问三用
- Zeno 专用内容资产闭环
- 把模糊词拆成可判断问题

### 3. 服务件

给客户交付或咨询使用。

标准：能进入 intake、检查清单、审核报告、咨询脚本。

候选升级对象：

- 签字前的问题翻译
- 签约前模糊问题追问清单
- 报价风险词库
- 装修报价风险洞察六层框架

### 4. 内容件

给外部表达使用。

标准：面向读者说人话，不暴露内部方法名。

候选升级对象：

- 旧观点卡
- 人工写稿素材包
- 网站文章母稿
- 小红书/公众号/短视频改写稿

## 第一阶段：收束总纲

目标：让所有方法都能挂到一张图上。

已建立主文件：

- [[Zeno判断资产系统总图]]

接下来要做：

1. 每次新增方法卡，先判断它属于总图哪一层。
2. 方法库索引中新增“总纲入口”。
3. Zeno OS 总首页中新增“判断资产系统”入口。
4. 避免再把所有方法都塞进 ZenoAIHome。

## 第二阶段：补齐 5 张核心方法卡

优先补这 5 张，因为它们是系统骨架：

1. [[商业判断方法/一问三用：把真实问题变成内容、产品和复利资产.md|一问三用]]
2. [[内容表达方法/语言校准：把伪概念变成可行动判断.md|语言校准]]
3. [[内容表达方法/内容资产转化纠偏：不要抄表面要吸收底层结构.md|内容资产转化纠偏]]
4. [[项目推进方法/Zeno专用内容资产闭环.md|Zeno 专用内容资产闭环]]
5. [[流量选题方法/爆款选题双轨素材膨胀法.md|爆款选题双轨素材膨胀法]]

补齐重点不是写长，而是补：

- 什么时候用
- 输入是什么
- 输出是什么
- 怎么判断质量
- 能转成什么服务/内容/Skill

## 第三阶段：打通 ZenoAIHome 服务链

目标：让方法库反哺现金流，而不是只成为知识库。

核心链路：

```text
报价风险内容
→ 发一行报价/上传报价单
→ 免费风险初筛
→ 付费报价审核
→ 匿名案例复盘
→ 风险词库 / 服务 SOP / 内容选题回填
```

需要补的资产：

- 报价审核 intake 问题表
- 报价审核报告模板
- 报价风险词 → 服务追问映射表
- 免费初筛结果话术
- 付费服务 CTA 话术
- 匿名案例复盘模板

## 第四阶段：升级 Agent 执行层

目标：把高频方法从 Obsidian 笔记升级成可执行流程。

优先升级方向：

1. **Hermes Skill**
   - 适合稳定、重复、可模板化的流程。
   - 例如：报价风险洞察、内容资产闭环、公众号素材包生成。

2. **dbskill 子流程**
   - 适合商业诊断、对标分析、内容判断、目标拆解。

3. **gstack 流程**
   - 适合网站 QA、转化路径检查、页面视觉和交互验证。

4. **cron 定时任务**
   - 适合每日情报、每周复盘、网站健康检查、内容资产提醒。

## 第五阶段：建立复盘回填

每个内容或服务动作完成后，至少记录 5 件事：

```text
发布/交付了什么？
目标用户是谁？
产生了什么反馈？
暴露了什么新问题？
应该更新哪张方法卡、服务 SOP 或内容选题？
```

建议位置：

- `22_运行记录与复盘/`
- 对应内容资产单元下的 `复盘.md`
- 对应方法卡的“复盘回填”区

## 30 天优化节奏

### 第 1 周：收束

- 总图入口写入总首页和方法库索引。
- 选出 5 张核心方法卡。
- 不新增大目录，避免系统继续发散。

### 第 2 周：补字段

- 给 5 张核心方法卡补齐标准字段。
- 把“读得懂的方法”改成“能执行的方法”。

### 第 3 周：接服务

- 围绕 ZenoAIHome 报价审核服务，做一张“风险词 → 追问 → 服务报告模块”映射表。
- 检查网站内容有没有对应 CTA。

### 第 4 周：回填和升级

- 选 1 个高频流程升级成 Skill 或固定 SOP。
- 用 gstack 检查网站转化路径。
- 做一次内容/服务复盘，把反馈写回方法库。

## 判断标准

优化成功不是文件更多，而是：

- 新资料能快速归类。
- 高价值资料能变成可复用资产。
- 方法卡能被人和 Agent 调用。
- 内容能自然指向服务。
- 服务反馈能反过来升级方法。

## 禁止事项

- 不为“看起来完整”新增一堆空目录。
- 不把所有 AI / 成长 / 美学内容都塞进 ZenoAIHome。
- 不把方法库变成模板化内容工厂。
- 不用 AI 一键替代重要最终表达。
- 不删除旧文件，优先归档。

## 下一步最小动作

从今天开始，所有新材料先问一句：

> 它能不能进入“输入 → 理解 → 迁移 → 资产 → 转化 → 回填”这条链？

不能进入，就暂存或丢弃；能进入，再决定写成方法卡、内容件、服务件还是 Skill。
