# ZenoAIHome 指标仪表盘模板

更新时间：2026-05-09

---

## 1. 使用原则

这份表不是为了做漂亮报表，而是回答 3 个问题：

1. 什么产品真的有人买。
2. 什么交付真的能复制。
3. 什么资产真的在变厚。

当前阶段不要追太多指标，只看能影响决策的指标。

---

## 2. 北极星指标

当前唯一北极星：

签约前判断产品是否形成稳定现金流，并持续沉淀结构化资产。

用 3 个数字联合判断：

1. 月度付费单量。
2. 单均毛利。
3. 单均有效资产沉淀数。

## 4. C 端验证指标

| 指标 | 定义 | 当前建议阈值 | 说明 |
|---|---|---:|---|
| 咨询到付费转化率 | 有效咨询中最终付费比例 | >= 8% | 低于此值，多半是产品口径或信任感有问题 |
| 报价快审复购/升级率 | 快审后升级更高服务的比例 | >= 20% | 低于此值，说明产品层级衔接不顺 |
| 单次交付毛利率 | 扣除时间成本后的毛利水平 | >= 60% | 低于此值，不适合继续靠人工放大 |
| 平均交付时长 | 单单从收材料到交付结束 | <= 2-4 小时 | 超过说明模板化不足 |
| 好评率 | 明确正反馈占比 | >= 85% | 低于此值要先修交付 |
| NPS | 是否愿意推荐 | >= 40 | 用于判断信任势能 |

## 5. B 端验证指标

| 指标 | 定义 | 当前建议阈值 | 说明 |
|---|---|---:|---|
| 访谈到试点转化率 | 被访谈 B 端进入试点的比例 | >= 20% | 低于此值，说明问题不够刚或产品说不清 |
| 试点付费率 | 进入试点的人里真正付费的比例 | >= 30% | 判断是否只是礼貌兴趣 |
| 试点完成率 | 试点过程完整参与比例 | >= 80% | 低于此值说明交付负担过重 |
| 模板复用率 | 一次交付中可复用模块占比 | >= 50% | 是能否做订阅的前置条件 |
| 续费/继续合作意愿 | 试点结束后愿意继续的比例 | >= 30% | 用于判断订阅可能性 |

## 7. 风险预警指标

以下任意 3 项连续两周异常，就该收缩或重做：

1. 咨询增加，但付费不增。
2. 客单价上升，但满意度下降。
3. 交付单量增加，但交付时长同步失控。
4. B 端访谈很多，但没有真实付费。
5. 案例越来越多，但模板复用率不上升。
6. 用户问题高度分散，说明定位开始发散。
7. 内容带来很多围观，却没有高质量线索。

## 9. 推荐阈值解释

### 可以继续加码

满足以下 4 条中的 3 条：

1. C 端转化率稳定达标。
2. 至少一个升级产品成立。
3. B 端出现明确付费样本。
4. 资产沉淀数量和质量持续增长。

### 应该先收缩

满足以下任意 2 条：

1. 主要收入仍完全靠创始人即时判断，模板化无进展。
2. 大量咨询但付费弱。
3. B 端强兴趣弱付费持续存在。
4. 共研、平台、合作谈得很多，但核心产品没跑通。

---

## 10. 建议的更新节奏

1. 每周五更新一次周度经营看板。
2. 每月最后一天更新一次月度决策表。
3. 每完成 10 单 C 端交付，就更新一次风险标签和案例库。
4. 每完成 3 单 B 端交付，就更新一次 SOP 复用率判断。

---

## 11. 当前阶段最重要的提醒

不要用总流量判断项目成败，要用以下顺序判断：

1. 有没有人付费。
2. 付费后满不满意。
3. 交付后能不能复用。
4. 复用后能不能扩张。

如果前两步都没站住，后面的平台化都只是想象。
