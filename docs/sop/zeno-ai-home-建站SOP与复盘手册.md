## 01 我为什么要做这个网站

不是装修公司官网。装修公司官网是给业务员发名片用的，第一目标是留电话。我不需要。

不是个人简历页。简历页是给面试官看的，更新频率低、信息死。我要的是每周都长出新东西。

它的真实身份是**一块自己的地**：

- 我每天的判断、写的字、做的工具、卖的服务，都先长在这里，再往别的平台分发。
- 公众号、小红书、视频号是租来的，平台改一次规则，所有积累一夜归零。
- 这块地是我的，链接结构是我的，文章是我的，数据库是我的，邮件列表是我的。

所以这个网站要同时承担四件事：

1. **内容根据地**——所有文章首发或归档在这里。
2. **工具承载体**——预算自测、报价指南、提示词库这种"有用的小东西"。
3. **服务转化口**——报价审核、预算咨询、AI 工作流咨询的入口。
4. **数字产品的货架**——后面要做的 PDF、清单、模板、课程都摆这里。

判断标准只有一条：**三年后还在不在、还能不能继续长。**

## 02 一开始的误区

按时间顺序，我自己掉过的坑：

### 误区 1：先追求"高级感"
最早花了不少时间调字体、调留白、调动效。结果上线后看后台数据：用户在首页停留 8 秒就走。原因不是不好看，是**没看明白这里能给他什么**。

教训：传统行业个人网站，**信息清晰 > 设计高级**。不是不要审美，是审美要服务于"看得懂"。

### 误区 2：页面太像展示站
首页一开始是大图 + 三段品牌故事 + "我们的优势"三栏。看上去像家装公司官网的小清新版。但我不是公司，我是一个人。

改完之后：首页第一屏只讲一句话——我是谁、我在写什么。下面直接是最新文章和工具入口。

### 误区 3：CTA 不清楚
最早每篇文章末尾就一句"如有问题欢迎联系"。这种 CTA 等于没有。用户读完一篇报价文章，最自然的下一步应该是**做一次自测**或**看完整指南**，而不是凭空联系陌生人。

后来五条业务线统一了 CTA 模板（见第 06 节）。

### 误区 4：资料库没有承担转化中枢
最早的 `/resources` 是一个文章合集，相当于第二个博客。完全浪费了它的位置。

资料库的真实角色应该是：**用户从任何文章进来后，能在这里找到下一步要做的事。** 它不是文章列表，是工具+清单+服务的导航台。

后来重排成四层：
- 第一层：装修前要先判断的（预算自测、报价指南）
- 第二层：装修中要落地的（合同模板、验收清单）
- 第三层：真实居住派的选择（材料判断、空间使用）
- 第四层：AI 辅助（提示词包、md2wechat 等工具）

### 误区 5：AI 功能容易先于真实用户路径
最早差点把 AI 聊天框做成首页 C 位、强提示用户问问题。后来意识到：

**用户来网站第一次不是来跟 AI 聊天的，是来确认"这个人靠不靠谱"的。** AI 是辅助，不是主角。

最终把 AI Chat Widget 收到右下角浮窗，进入门槛低、退出成本低，不抢主路径。详见第 08 节。

### 误区 6：想一次性把所有页面做完美
后来改成"**先把首页 + 1 篇支柱文章 + 1 个工具 + 1 个服务页**做到能看就上线"。剩下的边用边改。上线之后改一次和上线前调整一次，时间成本不一样，但学习速度天差地别。

## 03 核心目标重新定义

**当前阶段，网站第一目标不是直接成交装修服务。**

南宁本地装修是低频高客单决策，用户从看到我到成交，平均要走几周甚至几个月。指望网站直接转化是不现实的。

**第一目标是：让用户通过一个低门槛工具或资料，第一次体验到 Zeno 的装修判断力。**

具体落地：

- 第一次访问 → 读一篇文章或做一次预算自测
- 留下邮箱领资料 → 进入轻度信任池
- 看到我处理问题的方式 → 在适合的时候点开 `/services`
- 真有装修需求 → 走 `/contact`

转化漏斗按这个顺序排，每一步都要可衡量、可改：

| 阶段 | 指标 | 当前优先级 |
|---|---|---|
| 触达 | 自然搜索 + 平台引流到站访问量 | 高 |
| 体验判断力 | 工具完成率、文章阅读完成率 | 高 |
| 留资 | 邮箱订阅、资料领取 | 中 |
| 服务咨询 | `/contact` 提交、微信咨询 | 中 |
| 直接成交 | 报价审核、预算咨询订单 | 低（先不优化） |
| 装修服务签单 | 9800 起套餐 | 暂不指望网站直接成交 |

## 04 技术路线 SOP

写这一节不是炫技。是为了未来接手的人（或一年后的我）能快速理解为什么是这个组合。

### 技术栈与选型理由

| 选型 | 为什么是它 |
|---|---|
| **Next.js 14 (App Router)** | SEO 友好（SSR）、文件路由清晰、API Route 和页面在同一个仓库——一个人维护正好。 |
| **TypeScript** | 一个人开发久了一定会忘上下文，类型就是给未来的自己留的小抄。 |
| **Tailwind CSS** | 不想再开 CSS 文件来回切。Tailwind 把样式和结构放一起，改起来直观。 |
| **Vercel** | Next.js 官方部署，git push 自动上线，免费额度够个人站用。 |
| **GitHub** | 唯一真源。Vercel 只是消费者。 |
| **Supabase (Postgres) + Prisma** | 需要存订单、留资、笔记。Supabase 免费层够用，Prisma 让数据库改字段不痛苦。 |
| **NextAuth v5 (Auth.js)** | 后台登录、用户系统。配合 Prisma Adapter 一行接好。 |
| **Resend** | 发邮件。比 SendGrid 简单得多，对 Vercel 友好。 |
| **Claude Code** | 写代码、改代码、改文案、做重构的主力。比从零写快不止一倍。 |
| **DeepSeek API（OpenAI 兼容协议）** | AI Chat Widget 后端。便宜、中文好、有兜底回退。 |

### 关键约定

- **唯一真源在 GitHub**。本地、Vercel、Claude Code 都是消费者。任何"只在 Vercel 上改一下"的修改都视为临时，必须落到 git。
- **环境变量分两层**：`DATABASE_URL` `LLM_API_KEY` `RESEND_API_KEY` `NEXTAUTH_SECRET` 这些上 Vercel；本地用 `.env.local`，**`.env.local` 永远不进 git**。
- **DATABASE_URL 缺失也能 build**。Prisma 客户端做了 runtime stub（见 commit 76304f7），保证早期没接数据库时网站也能上线。
- **关键 admin 路由 force-dynamic**。避免 build 期触发数据库查询导致部署失败（见 commit b5a1855）。

### 一份完整的环境变量清单（敏感值仅占位）

```bash
# 数据库
DATABASE_URL="postgresql://..."

# NextAuth
NEXTAUTH_SECRET="..."         # openssl rand -base64 32 生成
NEXTAUTH_URL="https://zenoaihome.com"

# 邮件
RESEND_API_KEY="re_..."
RESEND_FROM_EMAIL="zeno@zenoaihome.com"

# AI 大模型（DeepSeek 兼容 OpenAI 协议）
LLM_API_KEY="sk-..."
LLM_BASE_URL="https://api.deepseek.com/v1"
LLM_MODEL="deepseek-chat"

# 站点
NEXT_PUBLIC_SITE_URL="https://zenoaihome.com"
```

### 本地开发到上线的最小动作链

```powershell
# 1. 拉最新
git pull origin main

# 2. 安装依赖（只在 package.json 改了之后）
npm install

# 3. 本地起服务
npm run dev   # http://localhost:3000

# 4. 改完后做三件事（详见第 09 节）
npx tsc --noEmit
npm run build
git push
```

## 05 页面结构 SOP

每一页只允许有一个第一目标。如果说不出来，就是没想清楚。

### 首页 `/`
**第一目标**：让陌生人 30 秒内决定要不要继续看。

承担：
- 一句话讲清"我是谁、我在写什么"
- 三个入口：看最新文章 / 了解我是谁 / 领实用资料
- 精选 8 篇能代表判断力的文章（不是最新，是最能代表）
- 工具入口（预算自测放在显眼位置）
- 服务入口（克制，不放价格不放促销）
- 底部 Footer 收纳所有链接

不承担：
- 不放装修流程图、不放服务三件套、不放"为什么选我"
- 不放任何一个 CTA 形式的弹窗、漂浮按钮（AI Chat 除外，因为它是工具）

### 文章页 `/blog/[slug]`
**第一目标**：让读者看完一篇文章后，自然地走向下一步——做工具、看下一篇、看服务、留邮箱。

五板块底部 CTA（这是 P0 改版的核心收益，commit ff1f088）：
1. 标签 + 上一篇/下一篇
2. 推荐相关文章（同主题地图下的 2-3 篇）
3. 推荐工具（按文章主题匹配，预算类配自测、报价类配指南）
4. 服务入口（弱化，一句话 + 链接到 `/services`）
5. 邮箱订阅（"想看后续，留个邮箱"）

不承担：
- 不放评论区作为转化口（评论是社区，不是漏斗）
- 不放分享按钮放大化（用户想分享自然会复制链接）

### 资料库 `/resources`
**第一目标**：所有文章读者的下一步在这里能找到。

四层结构（顺序就是装修生命周期）：
1. **装修前要先判断** —— 预算自测 / 报价指南
2. **装修中要落地** —— 合同模板 / 验收清单 / 主材判断
3. **真实居住派选择** —— 空间使用 / 材料判断 / 长期保养
4. **AI 辅助** —— 提示词库 / md2wechat / 公众号工具

每张卡片三件信息：是什么、解决什么问题、点进去做什么。

不承担：
- 不是文章合集（那是 `/blog`）
- 不放"全部资料"分类，要做选择

### 服务页 `/services`
**第一目标**：让真有需求的人快速判断"我适不适合"。

每个服务卡片必须写清：
- 谁适合（一句话画像）
- 谁不适合（一句话）
- 流程怎么走（3 步以内）
- 价格（明码标价，不引导加微信问价）
- 下一步链接（直接到下单页或 `/contact`）

当前四个服务：
- 报价单审核 ¥699/份
- 预算咨询 ¥399/次
- 真实居住派装修服务 ¥9800 起（仅南宁市区）
- AI 内容系统咨询 ¥1999/次

不承担：
- 不放案例展示墙（暂时没有可以公开的客户案例）
- 不放"成功故事"（编不出来不编）

### 关于页 `/about`
**第一目标**：让对内容产生兴趣的人，确认"这个人靠不靠谱、是不是真人"。

写法：
- 第一段：现在在做什么（不是从小聊起）
- 第二段：怎么走到现在（16 年装修 → AI → 一人公司）
- 第三段：现在的判断和价值观
- 第四段：联系方式 + 不接什么活

**不写"愿景使命价值观"。** 不写公司化的话。

### 文章底部 CTA 设计原则
- 永远不只放一个"加微信"
- 永远先给"再读一篇/做一次工具"这种**零成本下一步**
- 服务入口放在第三或第四位，不是第一位
- 邮箱留资放最后，门槛最低也最弱
- CTA 文案动词开头："试一下" "看看" "继续读"，不用"立即获取/马上解锁"

## 06 Zeno 文字心率规则

这套规则在 `赞诺内容资产系统/00_定位系统/` 已经有完整版，这里是写代码、写页面文案时的最小可执行版本。

### 标题
- 动词开头或问句开头，不用"关于…的几个思考"
- 不超过 14 个字
- 不用"终极/秘籍/避坑/必看"
- 示例：
  - ❌ 装修预算管理的几大要点
  - ✅ 装修预算，10 分钟体检
  - ❌ 报价单避坑终极指南
  - ✅ 报价单怎么看

### 副标题
- 一句话讲清这一页能给什么
- 不用排比、不用三联句、不喊口号
- 示例：
  - ✅ 10 个问题，看看你这次装修最可能在哪里超支。
  - ❌ 专业、权威、贴心，让你的装修无忧无虑。

### CTA 按钮
- 动词开头，3-6 个字
- 说清下一步是什么，不说"价值"
- 示例：
  - ✅ 开始自测 / 看完整指南 / 留个邮箱
  - ❌ 立即解锁 / 马上领取 / 一键拥有

### 服务说明
- 第一句说"谁适合"，不是说服务多好
- 第二句说流程
- 第三句说价格
- 不用"专业团队/资深顾问/一对一"

### 产品说明（PDF / 资料包）
- 写清楚它是什么（一份 PDF / 一张表 / 一段提示词）
- 写清楚解决哪个具体问题
- 写清楚要不要付钱、付多少
- 不写"价值 999 现价 9.9"

### 禁用词（必须替换）
- AI 腔：作为一个 AI / 综上所述 / 需要注意的是 / 值得一提的是
- 营销腔：闭眼入 / 全网最低 / 保姆级 / 干货满满 / 解锁
- 焦虑腔：90% 的人都不知道 / 现在不看就晚了
- 公文腔：随着…的发展 / 在当今时代 / 基于以上分析
- 数字夸张：100% 有效 / 省 X 万的秘诀 / 1 天搞定
- 模糊立场：仁者见仁 / 这个问题没有标准答案

### 推荐表达
- "我见过" "我审过的报价里" "在我做过的工地上"——用第一人称，不用"很多人"
- "大概率会…" "通常会…"——给判断但不给绝对承诺
- "如果…就…"——条件清晰
- 不知道就说"我没遇到过这个情况"

### AI 腔 vs Zeno 版本（示例）

**AI 腔**：
> 在装修过程中，预算管理是一个非常重要的环节。综上所述，合理规划预算可以帮助您避免后期超支。我们建议您从以下几个方面入手……

**Zeno 版本**：
> 装修预算最容易出事的不是单价，是漏项。我审过的报价单里，差不多每三份就有一份漏了一类大头——可能是水电改造，可能是垃圾清运，也可能是软装。先把这些口子堵上，再谈单价高低。

## 07 第一个获客工具 SOP：装修预算风险自测表

### 为什么先做自测表
四个原因：

1. **门槛最低**：不需要注册、不需要付费、不需要等待。
2. **直接展示判断力**：10 道题问完，结果页本身就是 Zeno 视角的输出。
3. **天然分享点**：URL 带答案，用户可以收藏、可以发给朋友、可以做完之后改一题再看。
4. **数据反哺**：哪些题被跳过、哪个维度高分多，直接告诉我用户最焦虑什么。

不做"PDF 留邮箱领"作为第一个工具。原因：留邮箱是**摩擦**，不是**体验**。第一次接触应该是体验，不是交换。

### 用户路径
```
首页/文章 → /tools/budget-risk
   ↓ 答 10 题
   ↓ 自动跳转
/tools/budget-risk/result?q1=...&q2=...
   ↓ 看 4 维度结果 + 主导风险解读
   ↓
[下一步入口]
   ├→ 看《报价单怎么看》指南
   ├→ 预约报价单审核服务
   └→ 留邮箱接收完整版预算模板
```

### 表单字段
10 道题，覆盖 4 个维度。题与维度的映射写在 `app/tools/budget-risk/questions.ts`。

| 维度代号 | 名字 | 关键判断点 |
|---|---|---|
| R1 | 漏项风险 | 水电、拆除、垃圾、软装、家电这类大头有没有都算 |
| R2 | 增项风险 | 报价描述够不够紧、有没有"按实际发生" |
| R3 | 决策风险 | 主材选择路径、决策权分散程度 |
| R4 | 时机风险 | 当前阶段还能不能改、签合同没 |

题型只用单选和多选，不用滑块、不用评分。理由：滑块在手机上很难精确，会引发"凭什么我是 7 分"的质疑。

### 风险维度评分
- 每个选项预设 4 个维度的分数（写死在 `questions.ts`）
- 答案累加 → `scoring.ts` 转成 low / mid / high 三档
- **不输出百分制**：百分制会引发"凭什么 75 分"的争论。三档清晰，留有解释空间。
- 主导维度 = 最大分；并列时按 R1 > R2 > R3 > R4（按真实损失大小排）

### 结果页逻辑
结构（从上到下）：
1. **顶部一句话判断**：根据主导维度生成。例如"你这次装修最可能在'增项'上出事"。
2. **四个维度卡片**：每个维度一句口语化结论 + 一段"为什么会这样"。
3. **针对主导维度的具体建议**：3-5 条可执行动作。
4. **三个下一步**：
   - 想看完整逻辑 → `/pricing/baojia-guide`
   - 想让我帮看具体报价 → `/services` 报价审核
   - 想收藏改进 → 复制本页链接

### CTA 设计
结果页上不放"立即购买"。三个下一步按"信息→服务→保存"的顺序排，意图越来越重。

特别注意：**结果页不强制留邮箱**。留邮箱降级成"想要 PDF 版完整结果，留个邮箱"——一个可选动作，不挡路。

### 如何承接报价指南 / 咨询 / 继续阅读
结果页底部三块卡片对应三种意图：

## 08 AI 聊天框接入复盘

### 为什么可以接入大模型
两个理由：
1. 装修问题里有大量"重复且边界清晰"的问题，比如"全包半包啥区别""预算怎么分块"，AI 用我的提示词答，质量稳定。
2. 24 小时在线。我自己睡觉也能给用户一个体面的回应。

### 为什么不要把它放在第一优先级
三个理由：

1. **AI 不能替我建立信任**。用户对"AI 对话"和"和 Zeno 对话"的认知是分开的。第一次接触如果只接触到 AI，反而会稀释我的人格感。
2. **AI 答错的成本高**。装修是高客单决策，AI 一句"建议这样做"出错，用户损失可能上万。所以提示词必须严格限定边界（见下文）。
3. **AI 是工具，不是产品**。这个站现在卖的是判断力，AI 只是判断力的延伸。

落地结论：**AI Chat Widget 放右下角浮窗**，主路径上不强提示。

### 当前实现方式
`app/api/chat/route.ts`：
- POST 接口，接收 `{ message, locale, history }`
- 优先调用 LLM（DeepSeek 兼容 OpenAI 协议）
- 写死的中英双语 System Prompt 注入"Zeno 助手"人格
- 浏览器端不直接持有 API Key，全部走 server

`components/AIChatWidget.tsx`：
- 浮窗、可关闭、可折叠
- 多轮上下文（最近 N 条传 history）
- 链接渲染：AI 输出 `→ /tools/budget-risk` 这种行，前端直接渲染成可点击链接

### 没有 API Key 时如何回退
这是一个被低估的工程细节。`route.ts` 里：

```ts
if (!process.env.LLM_API_KEY) {
  return NextResponse.json({ reply: fallbackAnswer(message, locale) })
}
```

`fallbackAnswer` 是一个关键词字典：命中"预算/报价/服务/AI/资料/联系"这些词，返回预写答案 + 站内链接。

好处：
- 本地开发不用接 Key 也能调界面
- 万一 Key 失效或欠费，用户看到的不是"500 错误"，而是一个简化版回答

### 系统提示词如何限制 AI 不乱说
关键约束写死在 `SYSTEM_PROMPT_ZH`：

- **使命**：帮用户看清问题，不是推销
- **说话方式**：禁用"赋能/打造/解锁"，不用 emoji，不用感叹号
- **知识范围**：只答装修、AI 实践、一人公司、网站本身四件事
- **边界**：
  - 不知道直说不知道
  - 不替用户做最终决定
  - 不替任何品牌推荐或贬低
  - 涉及法律、合同、人身安全 → 提醒找专业人士
- **不冒充 Zeno 本人**：要约见、要看具体材料 → 引导到 `/contact`
- **回复长度**：默认 3-6 句，清单类最多 5 条

### 后续如何升级为装修判断助手
路线图：

1. **现在**：通用 + 关键词回退。
2. **下一步**：把"已发布的高质量文章"做成 RAG 知识库，AI 回答时引用具体文章段落。
3. **再下一步**：接入"预算自测的结果数据"，用户做完自测后 AI 能针对结果给后续建议。
4. **更远**：训练一个真正的"Zeno 报价审核 mini-agent"，能让用户上传报价单截图初步分析。

## 一个装修人是怎么把自己的网站做出来的

我做装修做了 16 年。前 14 年没有自己的网站，全部生意来自工地推荐和朋友介绍。

去年我决定做一个属于自己的网站。不是公司官网，不是简历页，是一块**自己的地**。

下面是我从零开始走过的路，写下来给和我一样的传统行业从业者参考。

### 一、我为什么要做这个网站

我不是为了卖装修。

南宁本地装修是低频生意，一个客户从看到我到签单，平均要走几周甚至几个月。指望网站直接成交，不现实。

我做这个网站，是为了让一件事变成可能：**让陌生人在 30 秒内判断出我的判断力。**

如果他第一次看到我就觉得"这个人靠谱"，剩下的事就好办。如果第一次没建立这个印象，后面再多广告也没用。

所以网站要承担四件事：写字、做工具、放服务、卖小产品。

### 二、最早我犯过的几个错

**错 1：太追求好看。** 调字体、调动效、调留白，结果上线之后用户 8 秒就走了。不是不好看，是看不懂能给他什么。

**错 2：把首页做成展示墙。** 大图 + 三段品牌故事 + "我们的优势"。我不是公司，我是一个人。后来改成第一屏只讲一句话——我是谁、我在写什么。

**错 3：CTA 模糊。** 文章底下只写一句"如有问题欢迎联系"。等于没写。读者读完一篇报价文章，最自然的下一步是"做一次自测"，不是凭空联系陌生人。

**错 4：资料库变成第二个博客。** 把它做成文章合集是浪费。资料库的真实角色是**导航台**——读者从任何地方进来，都能在这里找到下一步要做的事。

**错 5：AI 功能放得太靠前。** 用户来网站不是来跟 AI 聊天的，是来确认这个人靠不靠谱。AI 是辅助。

### 三、我把网站的目标重新定义

**当前阶段第一目标：让用户用一个低门槛工具或资料，第一次体验到我的判断力。**

不是直接成交，是先让他**看到**。

具体路径：
- 第一次访问 → 读一篇文章或做一次自测
- 留邮箱领资料 → 进入轻度信任池
- 看到我处理问题的方式 → 在合适的时候点开服务页
- 真有需求 → 联系我

每一步都让用户多看一眼，不抢、不催、不焦虑。

### 四、我用了什么工具

不要被技术名词吓到。这套组合一个人能搞定。

- **Next.js**：建网站的框架。用它的人多，资料多。
- **Vercel**：把网站放到互联网上的地方。免费够用。
- **GitHub**：保存代码的地方。
- **Tailwind**：写样式更快。
- **Claude Code**：让 AI 帮我写代码、改代码、改文案的主力工具。
- **DeepSeek**：网站上 AI 助手用的大模型。中文好、便宜。

我不是程序员。这套东西我能用，是因为有 AI 帮我处理细节。我只需要把"我想要什么"说清楚。

### 五、每个页面只做一件事

我后来给自己定了一条死规矩：**每一页只允许有一个第一目标。如果说不出来，就是没想清楚。**

- **首页**：让陌生人 30 秒内决定要不要继续看。
- **文章页**：让读者看完之后自然走向下一步。
- **资料库**：让所有文章读者的下一步都能在这里找到。
- **服务页**：让真有需求的人快速判断"我适不适合"。
- **关于页**：让对内容感兴趣的人确认"这个人是不是真人"。

每页超过一个目标，用户就开始犹豫。犹豫就是流失。

### 六、我怎么写文字

我有一套自己的"文字心率规则"。简化版四条：

1. **动词开头**。"开始自测"，不是"立即解锁"。
2. **第一人称给判断**。"我审过的报价里…"，不是"很多人都…"。
3. **不知道就说不知道**。不编案例，不编数据。
4. **禁用一组词**：解锁 / 赋能 / 一站式 / 闭眼入 / 90% 的人都不知道 / 现在不看就晚了。

举个例子。

AI 腔的版本：
> 在装修过程中，预算管理是一个非常重要的环节。综上所述，合理规划预算可以帮助您避免后期超支。

我会改成：
> 装修预算最容易出事的不是单价，是漏项。我审过的报价单里，差不多每三份就有一份漏了一类大头。

谁在说话，听一句就听出来了。

### 七、我做的第一个工具：装修预算风险自测

为什么先做这个，不做别的？

四个原因：
1. **门槛最低**：不用注册、不用付费、不用等。
2. **直接展示判断力**：答完 10 题，结果页就是我的视角。
3. **天然分享点**：链接带答案，可以收藏、可以发朋友。
4. **反哺数据**：哪些题被跳过、哪个维度高分多，直接告诉我用户最焦虑什么。

我没有让它一上来就要邮箱。**第一次接触应该是体验，不是交换。** 留邮箱降级成结果页一个可选动作。

结果页给三个下一步，不强推：想自己琢磨 → 看完整指南；想让我帮看 → 预约审核；现在还不急 → 留邮箱按月收信。

让对的人看到对的下一步，比按钮颜色重要得多。

### 八、AI 聊天框我做了，但没放 C 位

我在网站右下角放了一个 AI 助手，能 24 小时回答装修常见问题。

但我没让它出现在首页中央。原因：

1. **AI 不能替我建立信任。** 第一次接触只接触到 AI，反而稀释我的人格感。
2. **AI 答错成本高。** 装修是高客单决策，AI 出错用户可能损失上万。
3. **AI 是工具，不是产品。** 我卖的是判断力，AI 只是判断力的延伸。

我给这个 AI 写了一段很严格的人设说明：禁用营销词、不冒充我本人、涉及合同法律提醒找专业人士、不知道就说不知道。

最重要的：万一 AI 服务出问题，它会自动回退到一个关键词字典，给用户一个体面的简化回答。**用户不该看到 500 错误。**

### 九、上线之前我会过一遍清单

每次改完，走这套流程：

- 本地起一遍服务，自己点一遍
- 类型检查 + 本地构建必须通过
- 提交描述写中文，一次只做一件事
- 推上去，看自动部署日志
- 部署完打开线上版本，**特别是用手机看一遍**
- 表单填一遍假数据走完
- 每个 CTA 点一遍确认链接对
- 出问题不要慌，部署平台一键回滚比改代码快

这套流程跑熟了之后，从改完到上线大概 10 分钟。

### 十、产品化怎么走

我给自己定的顺序是"先验证后扩张"。

**先做免费的小东西**：报价审核清单、预算分配模板、验收清单。门槛低，用来积累邮箱列表。

**再做低价产品**：报价避坑完整指南 39 块、AI 装修提示词包 99 块。验证用户愿不愿意为一份判断力付小钱。

**最后做服务**：报价审核 699 一份、预算咨询 399 一次、装修服务 9800 起、AI 工作流咨询 1999 一次。

我刻意不做的事，也写下来防止反复纠结：
- 不做抖音直播带货
- 不做高客单训练营
- 不接外包装修公司官网
- 不做全国服务，南宁先做透
- 不做自营建材电商

**不做什么，比做什么更定义你是谁。**

### 最后

这块地是我的。链接结构是我的，文章是我的，邮件列表是我的，工具是我的。

平台改一次规则，那些只在平台上积累的东西可能一夜归零。但这块地不会。

如果你也是一个做了很多年实体行业的人，正在想要不要给自己做个网站，我的建议是：

**做。但不要做装修公司官网那种。做一块自己的地。**

先把首页 + 一篇能代表你判断力的文章 + 一个小工具 + 一个明码标价的服务，做到能看就上线。剩下的边用边改。

上线之后改一次和上线前调整一次，时间成本不一样，但学习速度天差地别。
