# ZenoAIHome 网站结构宪法

## 1. 网站不是一层，是三层

ZenoAIHome 要同时承载三层内容，但三层不是平均用力。

| 层级 | 主要对象 | 主要任务 | 代表页面 |
| --- | --- | --- | --- |
| 第一层：装修业主前台层 | 正在装修或准备装修的人 | 快速判断问题类型，拿到抓手，决定要不要服务 | /, /start, /resources, /tools, /tools/budget-risk, /services/renovation |
| 第二层：同频读者层 | 愿意长期阅读的人 | 建立信任，理解 Zeno 的判断来源 | /about, /blog, /blog/[slug], /topics |
| 第三层：传统行业 AI 实践层 | 想把经验放大的同行 | 理解 AI 如何服务真实行业，而不是替代行业 | /services/ai-workflow, /tools/prompts, AI 相关文章 |

## 2. 首页只做一件事

首页不是站点总目录，也不是个人履历，也不是公司门面。

首页的职责只有一个：

让新访客 10 秒内知道，这站主要帮普通业主解决哪些装修判断问题，以及该从哪里进。

AI、个人事业系统、OPC、一人公司这些内容可以出现，但只能做副叙事，不能盖过前台问题入口。

## 3. 中文站的主路径

### 路径一：问题进入

/ -> /start -> /resources 或 /tools -> /services/renovation

这是主站最核心的路径。

### 路径二：文章建立信任

/blog 或 /topics -> /blog/[slug] -> /resources 或 /services

这是内容转化路径。

### 路径三：AI 精选路径

/services/ai-workflow 或 /tools/prompts -> AI 文章 -> AI 咨询

这是并行路径，不应和装修业主主路径混成一团。

## 4. 各页面的结构职责

| 页面 | 结构职责 | 不能做什么 |
| --- | --- | --- |
| / | 先帮用户认问题，再分流 | 不能先讲品牌故事和职业实验 |
| /start | 按问题、阶段、材料成熟度分流 | 不能写成长篇品牌文案 |
| /resources | 把资料和问题一一对应 | 不能变成资料仓库目录 |
| /tools | 先把问题缩小 | 不能堆工具，看起来像导航页 |
| /services | 先分装修判断和 AI 工作流 | 不能写成四个服务平铺销售页 |
| /services/renovation | 对普通业主的核心服务页 | 不能掺太多 AI 叙事 |
| /services/ai-workflow | 对同行的并行服务页 | 不能说成万能自动化 |
| /about | 解释为什么值得长期关注 Zeno | 不能写成简历或创业流水账 |
| /blog/[slug] | 讲透一个问题，并把下一步接上 | 不能只有观点没有去路 |
| /topics | 把文章串成连续阅读路径 | 不能只是标签聚合页 |
| /contact | 筛选沟通，不是“欢迎咨询”口号 | 不能重复服务页内容 |

## 5. 导航原则

顶层导航必须服务分流，不服务炫耀信息量。

推荐理解方式：

1. 首页：总入口。
2. 按问题进入：最强问题分流入口。
3. 工具与资料：自助抓手。
4. 专题或居住相关入口：建立长期阅读关系。
5. AI 工作流：第三层并行入口。
6. 找我帮你看：最终人工判断入口。
7. 关于：信任说明书。
8. 联系：放在页脚和具体页面，不必抢主导航注意力。

## 6. 页脚原则

页脚不是第二个首页。

页脚只做三件事：

1. 给出最基础的站点导航。
2. 统一品牌一句话说明。
3. 放稳定联系信息。

## 7. 英文页的结构定位

英文页不是中文站镜像。

它更适合做“精选层”，承担三个职责：

1. 讲清楚 Zeno 是谁，来自哪里。
2. 呈现少量最能代表站点气质的文章与观点。
3. 让海外或英文读者理解：这是一个从 frontline renovation experience 长出来的判断站，而不是 generic agency site。

所以英文页可以不完整复制中文问题入口、工具页和自测页，但不能写成标准企业服务介绍页。

## 8. /pro 的未来定位

/pro 不应该一开始就做成会员站首页。

它未来更适合承接下面这些内容：

- 39 元深度指南。
- 1499 签约前决策包的交付说明。
- 更完整的清单包、案例拆解、节点说明图。
- 付费工具增强版。
- 脱敏案例和进阶资料。

/pro 解决的是“已经有问题意识，也认可 Zeno 的判断，希望拿到更完整抓手”的那群人。

## 9. 结构上的红线

1. 不允许首页、服务页、About 页同时大篇幅讲“自由事业系统”。
2. 不允许装修主线页面被 AI 词汇压住。
3. 不允许资源页和工具页脱离产品路径，变成孤岛。
4. 不允许英文页继续往 generic services catalog 方向滑。
5. 不允许 /pro 先于中文主站基础叙事成熟而重投入。
