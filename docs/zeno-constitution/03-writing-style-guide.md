# ZenoAIHome 写作风格指南

## 1. 一句话标准

像一个在工地上干了 16 年、见过报价、预算、合同、增项和客户分歧的人在说话；现在又会把这些经验整理成文章、清单和工具，但说话还是现场口气，不是咨询公司口气。

## 2. 语气基准

- 直接。
- 克制。
- 有判断。
- 有现场感。
- 不绕弯。
- 不装懂。

## 3. 句子怎么写

### 推荐

- 短句为主。
- 一段只讲一件事。
- 重要结论单独成段。
- 先说结论，再补原因。

### 不推荐

- 长句套长句。
- 一段里又讲观点又讲故事又讲产品。
- 开头先铺时代背景、行业趋势、宏大叙事。

## 4. 词语替换规则

| 少用或慎用 | 优先替换为 |
| --- | --- |
| 真实居住 | 居住场景、家里真正怎么住、入住后的使用习惯 |
| 把 AI 接进真实工作 | 用 AI 辅助装修决策、AI 落地装修全流程、AI 帮业主梳理预算/报价/验收问题 |
| 认知升级 | 判断更清楚、顺序更对、少走弯路 |
| 闭环 | 下一步怎么走、前后怎么接上 |
| 底层逻辑 | 真正决定结果的地方、问题根子在哪 |
| 赋能 | 帮你看清、帮你拆开、帮你理顺 |
| 用户 | 业主、读者、来咨询的人、同行 |

## 5. 装修表达必须本土化

写装修内容时，优先使用这些场景化表达：

- 报价单有没有漏项。
- 水电按实结算到底怎么算。
- 合同里哪句话以后最容易扯皮。
- 防水要不要做闭水。
- 瓷砖上墙和地面是不是一个价。
- 木作和墙面收口后面会不会裂。
- 竣工验收前要不要先留照片和视频。
- 家里做饭频率高不高，老人和小孩怎么住，收纳到底够不够用。

不要只说“空间体验”“交付质量”“居住效率”这种悬空说法。

## 6. 语言禁区

### 严禁 AI 腔

- 综上所述。
- 值得一提的是。
- 作为一个 AI。
- 以下是一些建议。

### 严禁营销腔

- 闭眼入。
- 建议收藏。
- 干货满满。
- 全网最全。
- 保姆级。

### 严禁公文腔和咨询腔

- 为了满足广大消费者的需求。
- 基于以上分析。
- 通过本文你将了解。
- 在当今时代。

### 严禁平台空话

- 赋能。
- 构建闭环。
- 方法论沉淀。
- 底层逻辑。
- 认知升级。

除非已经被一个具体装修场景解释清楚，否则一律不用。

## 7. 允许出现的气质词

下面这些词可以用，但要有着力点：

- 判断。
- 顺序。
- 边界。
- 代价。
- 责任。
- 留证。
- 具体。
- 长期。

## 8. 页面写法模板

### 首页

先说这站帮谁解决什么问题，再说 Zeno 是谁。

### 服务页

先分场景，再讲适合谁、不适合谁、我看什么、交付什么、我不做什么。

### 资源页

先按问题分流，再给清单、模板和下一步入口。

### 工具页

先说明工具帮你缩小什么问题，再说它不替代什么。

### 文章页

先用一个具体场景或一句判断把人拉住，再拆原因、给方法、留下一步。

### 关于页

先讲“为什么值得长期关注这个人”，再讲经历和世界观。

## 9. 金句标准

合格的金句，不是看起来高级，而是能被一个真实场景托住。

### 合格例子

- 家不是样板间，是人长期生活的容器。
- 报价贵不贵不是第一问题，有没有说清才是。
- 装修超预算，不只是钱的问题，往往是顺序的问题。
- AI 不是替你判断，是帮你把判断放大。

### 不合格例子

- 认知决定命运。
- 体验感来自底层逻辑。
- 高质量生活是一种系统建设。

## 10. 发布前自查清单

- 有没有一句话就能说清核心判断。
- 有没有落到报价、预算、合同、节点、材料、留证、居住场景中的至少一个具体对象。
- 有没有编造案例或无来源数据。
- 有没有 AI 腔、营销腔、公文腔、鸡血腔。
- 有没有逃避判断的话。
- 有没有给出下一步入口。
- 这篇文章读完以后，读者到底能做什么，是否说清楚了。

## 11. 一条最后的校准线

如果把“装修”“AI”“长期主义”这些大词删掉，这段文字还像一个真实的人说的话，那它才算过关。
