# AGENTS.md｜Zeno OS 协作规则

> 本文件给 Hermes / Codex / Claude 等 Agent 读取。它定义这个知识库的边界、写入规则和禁止事项。

## 1. 系统定位

这是 Zeno OS：个人认知、内容表达、流量选题、商业项目和 Agent 协作的本地资产系统。

它不是单纯的装修网站资料夹，也不是 AI 一键写稿工厂。

## 2. 项目层级

- `Zeno OS`：总系统。
- `06_项目库/ZenoAIHome`：装修报价风险、签约前咨询、网站/服务项目。
- `06_项目库/OPC一人公司AI培训`：面向各行各业的一人公司 AI 使用与经验资产化项目候选。
- `07_方法论与可调用方法库`：可复用方法、判断框架、操作法。

处理资料前先判断它属于哪一层，不要把所有内容都硬迁移到装修项目。

## 3. AI 的角色

AI 可以做：

- 检索旧观点；
- 整理资料；
- 提炼机制；
- 做素材膨胀；
- 提出反方和新角度；
- 生成索引、清单、方法卡、SOP；
- 辅助标题、开头、结构检查。

AI 不应默认做：

- 一键生成重要逐字稿；
- 编造真实案例；
- 用模板化内容替代 Zeno 的表达审美；
- 把所有流程包装成狭义 AI workflow；
- 删除文件或大规模迁移，除非任务明确要求。

## 4. 默认写入规则

收到外部资料/截图/链接/灵感时，默认按以下顺序处理：

1. 判断资料价值：高 / 中 / 低 / 丢弃。
2. 判断归属层级：认知、内容、流量、商业、项目、方法、故事、暂存。
3. 高价值资料进入 `00_输入收件箱` 或对应项目目录。
4. 可复用流程进入 `07_方法论与可调用方法库`。
5. 和 ZenoAIHome 直接相关的装修/报价/签约资料，进入现有 ZenoAIHome 资产路径或 `06_项目库/ZenoAIHome`。
6. OPC / AI 培训相关资料进入 `06_项目库/OPC一人公司AI培训`。
7. 每次重要写入后，在 `10_运行记录与复盘` 记录。

## 5. 内容原则

- 最终表达由人完成。
- 方法卡要短、清楚、可调用。
- 面向读者的文章要像人话，不要暴露内部工具语言。
- 观点是 Zeno 的，AI 负责整理和辅助，不替代判断。
- 可以多方法、多角度、多路径，但不要只剩一个万能模板。

## 6. 文件安全

- 不要删除文件，优先归档到 `90_归档`。
- 不要移动 `.obsidian/` 配置文件。
- 不要改网站代码目录，除非任务明确要求。
- 对外发布稿、内部方法卡、原始资料要分开存放。
