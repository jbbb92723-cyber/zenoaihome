# Coze + 飞书自动化内容监控系统说明

> 来源：用户转述/补充。归属：Zeno OS 内容雷达系统参考素材。价值：高。用途：完善 Zeno 内容雷达系统 v1 的字段、工作流、异常处理和拆解 prompt 设计。

## 原始材料摘要

这是一套基于 Coze + 飞书搭建的自动化内容监控系统，通过五层架构实现从数据采集到爆款拆解的全流程自动化。

## 系统核心价值

### 1. 数据资产沉淀

- 2000+ 视频数据结构化存储。
- 增量计算精确到个位数。
- 历史数据可追溯对比。

### 2. 智能分析能力

- 大模型自动拆解爆款内容。
- ASR 转写口播文案。
- 多维度爆款指数计算。

### 3. 工作流自动化

- 定时抓取博主更新。
- 自动去重与数据清洗。
- 异常熔断机制保障稳定性。

### 4. 人机协作设计

- 飞书群作为交互入口。
- 多维表格作为数据中心。
- 云文档自动归档分析报告。

## 典型工作场景

```text
凌晨 2 点自动启动抓取
→ 早上 9 点推送战报到群
→ 运营标记待拆解视频
→ 系统生成完整拆解文档
```

## 技术栈组合

- 前端：飞书多维表 + 群机器人
- 中台：Coze 工作流 + 插件
- AI 层：大模型分析 + ASR 转写

## 核心创新

- 将抖音数据 API 转化为可操作业务洞察。
- 建立内容资产的持续沉淀机制。
- 实现人工判断与自动化执行的配合。

## 完整搭建教程包含

1. 飞书多维表字段设计规范。
2. Coze 工作流配置模板。
3. 异常处理 checklist。
4. 爆款拆解 prompt 库。

## 对 Zeno OS 的迁移价值

这套系统最值得迁移的不是“抖音爆款监控”，而是：

```text
内容数据结构化
→ 增量计算
→ 历史对比
→ 定时战报
→ 人工标记待拆解
→ AI 自动拆解
→ 云文档/Obsidian 资产沉淀
```

## 对 Zeno 内容雷达系统 v1 的补充点

应补入：

- 增量计算字段，不只记录静态点赞收藏。
- 历史快照表或快照字段，方便追溯对比。
- 爆款指数计算，而不是只看单一指标。
- 异常熔断机制：抓取失败、接口异常、重复数据、限流。
- 人工标记字段：运营/Zeno 手动标记“待拆解”。
- 拆解文档归档字段：云文档链接 / Obsidian 文件路径。

## 初步判断

价值等级：高。

原因：这套系统验证了“现有 SaaS 工具 + AI + 人工判断”能搭建轻量业务系统，适合作为 Zeno 内容雷达系统的工程参考。但 Zeno 版本必须保持：不追爆款外壳，重点监控真实需求、痛点、报价风险和可转化服务机会。
