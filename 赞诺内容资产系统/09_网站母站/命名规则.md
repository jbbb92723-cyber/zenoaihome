# 命名规则

> 统一命名，避免后面检索、同步、发布、引用出错。

## 一、文件命名总原则

- 公开文件尽量用中文，便于人读
- 需要作为 URL、图片文件名、slug 的内容用英文小写或拼音小写
- 单词之间用 `-` 分隔
- 不用空格、不用下划线、不用混合大小写
- 同一主题只保留一个正式发布名

## 二、文章命名

### 知识库内部
可以使用中文标题：

```text
为什么装修预算总超.md
家不是样板间.md
从工地看世界.md
```

### 网站文章
网站文章建议用最终 slug 对应的中文标题文件，或按你现有结构继续保持中文文件名，但必须统一一套规则。

推荐：

```text
01_为什么装修预算总超.md
02_报价单真正该怎么看.md
03_实住派装修究竟反对什么.md
```

## 三、图片命名

### 原则
- 全部小写
- 英文或拼音
- 用 `-` 连接
- 明确用途

### 示例

```text
cover.webp
quote-checklist-preview.webp
budget-template-preview.webp
renovation-judgment-proof.svg
```

### 文章配图建议

```text
public/images/articles/<article-slug>/cover.webp
public/images/articles/<article-slug>/scene-01.webp
public/images/articles/<article-slug>/scene-02.webp
```

## 四、视频命名

视频不直接进网站仓库。命名只用于知识库备份和封面：

```text
video-01-cover.webp
video-01-notes.md
video-01-source.mp4
```

实际播放靠外部平台链接，不靠本地大视频文件。

## 五、资料与服务页命名

### 资料
```text
装修预算避坑资料包.md
装修报价审核清单.md
实住派装修自查表.md
AI内容创作提示词包.md
```

### 服务页
```text
装修报价单审核服务.md
装修预算咨询.md
真实居住派装修服务.md
AI内容系统咨询.md
```

## 六、骨架与成品区分

- `*_骨架.md` = 结构草案
- 正常文件名 = 成品稿或主稿
- `*_复盘.md` = 发布后总结
- `README.md` = 目录说明
- `说明.md` = 该目录用途说明

## 七、不要这样命名

- 不要用 `final-final-2`
- 不要用 `新建文档 1`
- 不要用纯英文缩写但自己后面看不懂
- 不要同时保留多个正式版本而不标注用途

## 八、落地优先级

先统一这三类：

1. 文章文件名
2. 图片文件名
3. 网站母站目录中的说明文件名
