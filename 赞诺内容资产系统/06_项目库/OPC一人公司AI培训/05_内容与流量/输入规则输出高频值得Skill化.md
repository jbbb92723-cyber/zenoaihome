## 原始输入

输入 + 规则 + 输出 + 高频 = 值得 skill 化。

## 一句话提炼

不是所有经验都值得封装成 skill，只有那些输入稳定、规则明确、输出可验收、并且高频发生的任务，才值得 skill 化。

## 核心判断

这条可以作为 Zeno OS / OPC 的 skill 化底层判断。

很多人一接触智能体、skills、工作流，就想把所有东西都封装起来。

但真正值得封装的，不是“看起来复杂”的任务，而是“反复出现，并且已经有稳定结构”的任务。

如果输入混乱、规则不清、输出不可验收，只是因为看起来高级就去做 skill，本质上是在把混乱自动化。

## 四个条件

### 输入

每次任务开始时，材料类型要相对稳定。

比如：一段外部观点、一篇文章、一个报价单、一个 AI 工具课程、一个待判断的行业方向。

### 规则

处理这类输入时，有明确判断标准和步骤。

比如：价值判断、归属判断、边界判断、输出格式、是否需要人工确认。

### 输出

处理完之后，必须有明确产物。

比如：方法卡、观点卡、诊断报告、风险清单、对标分析、内容生产单。

### 高频

这个任务要反复发生。

如果一年只发生一次，写笔记就够了；如果每周都发生，就值得方法卡化；如果每天都发生，就值得 skill 化。

## 和 OPC 的关系

OPC 不应该只教别人“做智能体”，而要教他们判断：

> 什么东西值得封装，什么东西不值得封装。

真正适合封装成 skill / 智能体的，通常是专业人士已经反复做过很多次的判断。

skill 的价值不在提示词表面，而在背后的专业判断：

- 输入识别；
- 判断规则；
- 输出验收；
- 高频复用；
- 边界处理。

## 和 Zeno OS 的关系

Zeno OS 里很多动作已经符合 skill 化条件：

- 外部观点收进知识库；
- OPC 观点沉淀；
- 方法卡生成；
- 内容资产生产单；
- 旧观点检索；
- 素材膨胀；
- 报价风险初筛；
- AI 工具/课程价值判断。

这些不是一键内容生产，而是把高频判断动作封装起来，给 Zeno 节省注意力。

## 内容可用角度

### 角度 1：不是所有任务都值得做成智能体

输入不稳定、规则不清、输出不可验收的任务，做成智能体只会制造混乱。

### 角度 2：skill 的本质是稳定判断的封装

真正值钱的不是提示词，而是背后的专业规则。

### 角度 3：高频任务才值得工程化

低频任务写笔记，高频任务做 skill。

### 角度 4：不要把混乱自动化

自动化之前，先把输入、规则、输出讲清楚。
