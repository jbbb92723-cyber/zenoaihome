# 网站媒体使用说明

> 给非程序员看的版本 — 告诉你图片和视频应该放哪里、怎么加、怎么改。  
> 最后更新：2026-04-19

---

## 一、网站图片放哪里

网站的图片放在这个目录里：

```
Zeno-Code/zeno-site/public/images/
```

按用途分了几个子目录：

| 目录 | 放什么 |
|---|---|
| `public/images/brand/` | 头像、品牌 logo、签名图 |
| `public/images/articles/` | 文章封面图和配图 |
| `public/images/topics/` | 各专题的配图 |
| `public/images/resources/` | 资料预览图 |
| `public/images/services/` | 服务相关图片（可选） |

> 注意：放进去的文件名必须是英文或拼音，不要用中文，不要有空格。

---

## 二、原图和网站压缩图的区别

| | 原图 | 网站用图 |
|---|---|---|
| 放在哪里 | `Zeno-Content/赞诺内容资产库/09_网站母站/媒体资产/图片原图/` | `Zeno-Code/zeno-site/public/images/` 对应子目录 |
| 体积 | 一般 2MB～20MB | 压缩后 50KB～200KB |
| 格式 | 相机 RAW、JPG 原图 | webp（首选）或 jpg |
| 作用 | 备份备用，以后重新处理 | 直接被网站调用展示 |

**原图绝对不要直接放进 `public/images/`，会导致网页加载极慢。**

---

## 三、推荐图片格式

| 格式 | 使用场景 | 说明 |
|---|---|---|
| `.webp` | 所有照片类图片 | 体积最小，首选 |
| `.jpg` | 备用，部分工具不支持 webp 时 | 压缩到 80% 质量 |
| `.png` | 需要透明背景的图片 | 比 jpg 大，非必要不用 |
| `.svg` | logo、图标、纯线条图 | 矢量图，任意缩放不失真 |

---

## 四、推荐图片尺寸

| 用途 | 推荐尺寸 | 比例 | 文件大小上限 |
|---|---|---|---|
| 首页/文章作者头像 | 200×200px 或 400×400px | 1:1 | ≤ 80KB |
| 文章封面图 | 1200×525px | 16:7 | ≤ 200KB |
| 文章列表封面图 | 800×350px | 16:7 | ≤ 150KB |
| 资料预览图 | 640×400px | 约 16:10 | ≤ 120KB |
| 专题配图 | 800×400px | 2:1 | ≤ 150KB |
| 品牌 Logo | SVG 矢量格式 | 任意 | ≤ 20KB |

> 不知道怎么压缩？用 [Squoosh](https://squoosh.app)（免费在线工具），上传图片，选 webp 格式，导出即可。

## 五、视频放哪里

**视频不放在网站代码里，放到外部平台：**

- **B站**（首选，中文内容最佳）
- **视频号**（微信生态传播）
- **YouTube**（境外流量 + AI 引用友好）
- 腾讯云 COS / Cloudflare R2（私有托管，需技术支持）

当前网站没有启用站内视频数据文件。发布后先在内容台或文档中保存视频链接和嵌入地址；等需要在网站展示视频时，再新增对应的数据文件和前台组件。

---

## 六、为什么大视频不要放 public/videos

1. **仓库体积爆炸**：一个视频文件几十到几百 MB，Git 仓库不适合存这类大文件，推送到 GitHub 会非常慢，甚至失败。

2. **Vercel 有文件大小限制**：单个文件超过 100MB 就无法部署。

3. **用户体验差**：没有 CDN 加速的视频，加载会很卡，严重影响体验。

4. **流量费用高**：Vercel 免费套餐的带宽有限，视频流量会很快超出，可能产生额外费用。

**结论：视频永远走外部平台，网站只保存链接。**

## 七、如何新增一张文章封面图（一步步）

1. 准备好图片，用 Squoosh 压缩为 webp 格式，控制在 200KB 以内。

2. 按文章 slug 命名文件夹，把图片放进去：
   ```
   Zeno-Code/zeno-site/public/images/articles/
   └── zhuangxiu-yusuan-weishenme-zongchao/
       └── cover.webp
   ```

3. 打开 `Zeno-Code/zeno-site/data/articles.ts`，找到对应文章，填写 `coverImage` 和 `coverAlt`：
   ```typescript
   coverImage: '/images/articles/zhuangxiu-yusuan-weishenme-zongchao/cover.webp',
   coverAlt: '装修预算为什么总超 — 文章封面图',
   ```

4. 保存文件，提交到 GitHub，Vercel 会自动重新部署。

5. 部署完成后，访问文章页面，封面图会出现在标题下方。

---

## 八、如何在 articles.ts 里绑定图片

找到对应文章的数据，把 `coverImage` 从空字符串改为真实路径：

**改之前：**
```typescript
coverImage: '',
coverAlt: '文章封面图',
```

**改之后：**
```typescript
coverImage: '/images/articles/你的文章slug/cover.webp',
coverAlt: '你的图片描述（写清楚图片内容，对搜索引擎有帮助）',
```

> 路径开头的 `/` 不要漏掉，代表从网站根目录开始找文件。

---

## 九、如何新增一个视频链接

1. 把视频发布到 B站（或 YouTube 等平台）。

2. 复制视频的：
   - 主页链接（普通分享链接）
   - 嵌入代码里的 `src` 地址（点分享 → 嵌入代码 → 找 `<iframe src="...">`）

3. 等网站需要展示视频模块时，再新增数据文件和前台组件。

4. 不要把原始大视频提交进仓库。

---

## 十、部署后如何修改图片和视频

**修改图片：**
1. 在本地把新图片放到 `public/images/` 对应目录
2. 更新 `data/articles.ts`（或 `topics.ts`、`resources.ts`）里的图片路径
3. 用 Git 提交到 GitHub → Vercel 自动重新部署

**修改视频链接：**
当前没有站内视频模块。先在外部平台或内容台维护视频链接；需要展示时再接入数据源。

**完全不需要：**
- 重新安装任何软件
- 在 Vercel 后台手动操作
- 重新配置域名

**只要：本地改文件 → 提交 GitHub → 等 Vercel 自动部署（通常 1-3 分钟完成）**
