## 一、UI/UX 改版

### 1.1 核心诊断结论

来自三轮设计评审（拉姆斯 / 塔夫特 / 诺曼 / 原研哉）：

| 诊断 | 严重程度 | 整改方向 |
|---|---|---|
| 七个角色挤在一个网站上，示能混乱 | 🔴 致命 | 首页首屏只回答一个处境问题，其余退入二级 |
| AI ChatWidget 被锁在后台（ConditionalAIWidget 限制 + 无 LLM_API_KEY） | 🔴 致命 | 取消路径限制，配置 API key，升级为情境预判入口 |
| 装饰元素超过内容需求（decision-scan 动画、冗余 CSS 类） | 🟡 中等 | 砍掉无信息价值的装饰，保留功能性视觉信号 |
| Bone Paper / Warm Sand 亮度差不足 | 🟡 中等 | 不急于改色，先观察上线后反馈 |
| 100+ 篇文章排版呼吸感不足 | 🟡 中等 | 增加阅读留白，引入「处境→子问题」索引 |

### 1.2 首页重构：三层叙事（三屏）

**与初版的关键变化（审计修正）**：
- 留白不独立成屏，融入第一屏底部让时间线自然呼吸
- 四个按钮统一为「任务目标」维度，B端用工长入口用视觉区隔（细线或不同底色）分开
- AI 在处境选择**之前**介入——帮用户判断自己属于哪个处境，不在按钮之后拦截
- 信任证据需要真实文章做锚点——个人叙事内容（Phase 0）必须先于首页上线

### 1.4 AI 交互层改造

| 改动 | 现状 | 目标 |
|---|---|---|
| ConditionalAIWidget 限制 | 仅 /notes 和一个博客页 | **全站开放**。所有核心页面出现浮动 AI 入口 |
| 聊天 API | 无 LLM_API_KEY，正则关键词回退 | **配置 DeepSeek API**。温度 0.4，max_tokens 600 → 1200 |
| ChatWidget 角色 | 路由工具（"先分流再解决问题"） | **情境预判入口**：用户到达任何页面，AI 根据当前页面主题主动提供 3-4 个快捷问题 |
| AI 在首页 | 无 | 首页首屏的处境选择本质上是 AI 引导的第一步——点击「找装修公司」→ AI 介入问 3 个关键问题 → 给出结构化判断，不是自由聊天 |

## 二、商业链路审计

### 2.1 现状与决策

当前 `commercial-ladder.ts` 只有装修服务（免费诊断 → 报价审核 → 节点顾问 → 床垫）。2026-07-14 审计后决策：**不新增 AI 培训产品。AI 不是产品，是 Zeno 做所有事的方法。** OPC 社群是独立产品，但不由 commercial-ladder.ts 承载——它是付费社群（知识星球），网站只做入口和门面。

### 2.2 产品梯子（最终版）

```
─────── 信任入口（免费）───────
  居住诊断 · 报价初筛 · 预算工具 · 检查清单
  → 目标：任何人零门槛体验 Zeno 的判断力

─────── 装修服务（付费）───────
  ¥2,500 报价审查 · ¥2,000+ 节点顾问 · ¥299 床垫
  → 目标：高客单价，深度服务，口碑传播

─────── OPC 社群（付费，独立产品线）───────
  OPC·同行有你 —— 知识星球承载
  网站角色：入口页（信条/规则/成员故事）→ CTA 跳转知识星球
  → 目标：复购引擎 + 转介绍网络
```

### 2.3 C 端转化漏斗

```
流量来源
├── AI搜索（GEO）→ "装修报价怎么看" → 文章 → 免费工具 → 付费审查
├── 知乎/小红书 → 内容片段 → 网站 → 居住诊断 → 付费服务
└── 朋友推荐 → 直接搜索 "Zeno" → 首页 → 处境选择 → 付费服务

转化节点
  到达首页 → 处境选择（AI辅助判断） → 免费工具使用
    → 结果页 CTA → 付费服务页 → 下单
    → 或 → 了解OPC社群 → 跳转知识星球

当前卡点：
- 首页没有处境按钮（只有模糊的CTA"先做诊断"）
- 免费工具结果页的付费CTA不够强
- OPC社群没有独立的网站入口页
```

### 2.4 B 端（工长）路径

```
工长到达 → 首页点击「我是工长」→ B端入口页
  → 了解机制（不是广告平台，是判断匹配）
  → 提交申请（项目照片 + 资质 + 自述）
  → Zeno 人工审核 + AI 初筛
  → 通过 → 建立工长档案 → 进入匹配池
  → 匹配到业主 → 平台内沟通 → 签约
  → 交付后评价 → 评价影响后续匹配权重

当前状态：B端路径完全不存在。没有页面、没有流程、没有数据模型。
```

### 2.5 定价与价值匹配

| 产品 | 定价 | 价值论证 | 状态 |
|---|---|---|---|
| 报价审查 ¥2,500 | 已确定 | 一次审查可能省 5-50 万增项 | ✅ 在售 |
| 节点顾问 ¥2,000+ | 已确定 | 全程 6-8 个节点，折合单节点 ¥250-330 | ✅ 在售 |
| 床垫 ¥299 起 | 已确定 | 自有选品，去中间商 | ✅ 在售 |
| OPC 社群 | 知识星球定价 | 同行网络 + 方法论 + 能量圈 | ✅ 在售（网站做入口） |

## 四、后台实用化

### 4.1 当前后台功能审计

后台共 16 个路由目录，7 个功能模块：

| 模块 | 功能 | 判断 |
|---|---|---|
| 数字大屏 | KPI dashboard + 转化漏斗 + 快捷面板 | ✅ 保留。核心运营视图 |
| 用户运营 | users / members / orders | ✅ 保留。刚需 |
| 内容与服务 | content / notes / services / living-diagnoses / pipeline | 🟡 内容管线 + pipeline 可能重叠，考虑合并 |
| 项目交付 | projects（7节点 + 照片 + 笔记） | 🟡 当前主要面向内部项目管理。如果 B 端工长要做交付管理，需要重构 |
| AI 工具 | ai-reply（RAG + 模板库） | ✅ 保留。但需要从「客服回复」扩展为「判断辅助」 |
| 营销工具 | redeem-codes / coupons / resources-claims | 🟡 优惠券和兑换码功能重叠，考虑合并 |
| 系统 | events / logs | ✅ 保留。合规 |

### 4.2 精简与增强

**可精简**：
- `pipeline`（内容管线）和 `content`（内容管理）→ 合并为一个内容工作台
- `coupons` 和 `redeem-codes` → 合并为营销码管理
- `services` 和 `living-diagnoses` → 合并为「服务请求」统一视图

**需增强**：
- **服务请求工作流**：当前是 4-tab（新请求 / 处理中 / 已完成 / 已关闭）。增加「AI 自动分类 → 优先级排序 → 智能路由（装修判断 / 社群相关）」的智能路由
- **工长管理**：新建。B 端入驻审核、工长档案、匹配记录、评价管理。当前不存在
- **AI 回复工具**：从「客服回复」升级为「判断辅助」——不是帮 Zeno 写回复，是帮 Zeno 更快地做出判断（结构化输入 → AI 对比历史案例 → Zeno 确认 → 输出）

### 4.3 后台优先级

| 优先级 | 改动 | 理由 |
|---|---|---|
| 🔴 P0 | 服务请求工作流升级（AI 智能路由） | 直接影响 C 端转化效率 |
| 🟡 P1 | 合并重叠模块（pipeline+content, coupons+redeem-codes） | 减少维护负担 |
| 🟡 P1 | 新建工长管理模块 | B 端平权的后台基础 |

## 五、分阶段执行路线图（审计修正版）

### Phase 0：内容与到达路径（本周，先于首页上线）

```

  原则：每篇至少一个AI编不出来的具体细节（地点/金额/人名/时刻）

□ 知乎+小红书外部引用启动（10篇日历的前3篇）
□ 配置 LLM_API_KEY（聊天 API 从正则回退升级到 DeepSeek）
```
