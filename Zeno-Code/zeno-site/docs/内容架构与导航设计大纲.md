# ZenoAIHome 内容架构与导航设计大纲

> 2026-07-05 定稿。2026-07-05 v2 修正。
>
> 网站七个角色：背书案例、内容中台、根据地、名片、商业用途、IP、培训讲台。
>
> **核心定位修正：AI 培训是商业产品，装修是案例根基。网站本身就是 PPT。**

---

## 一、基本定位

### 网站的逻辑

```
装修是案例层  →  "这个人真干过，他的判断有分量"
AI 是产品层  →  "他的方法论我能学，也能用在其他行业"
培训是商业层  →  "请他来给我们讲一场"
```

**网站讲述的故事线：**
> 一个干了 16 年装修的人，用 AI 把自己从头做了一遍。他的方法对任何传统行业都管用。这个网站，就是他站在台上时翻的 PPT。

### 网站七个角色（修正版）

| 角色 | 体现 |
|---|---|
| **培训讲台** | 整个网站就是 PPT。首页是封面，装修美学是案例章节，AI知识库是方法论章节，关于我是结尾 |
| **背书案例** | 装修美学里每一篇文章都在说"这个人真干过" |
| **商业用途** | AI 培训是主营收入，咨询服务和 OPC 社群是补充 |
| **IP** | "传统行业 × AI 实战"这个定位 = 赞诺 |
| **内容中台** | 网站是母版，分发到公众号/小红书/视频号 |
| **根据地** | .com 永远在，不被平台算法限流 |
| **名片** | 发给潜在培训客户，不需要补充解释 |

## 五、核心框架：「传统行业 × AI 五步法」

培训时的主讲纲。每一步都用装修案例证明，再抽象为通用方法。

```
        装修案例层                  AI 方法论层
      （证明我干过）            （任何传统行业都能用）

Step 1 │ 看清生活              用 AI 把你的经验结构化
       │ 空间方案与审美          "你脑子里攒了十年的东西，
       │                         AI帮你三天整理成体系"

Step 2 │ 看懂材料              用 AI 做信息对比和判断
       │ 材料与工艺判断          "参数对比、话术还原、风险识别"

Step 3 │ 选对人                用 AI 辅助沟通和管理
       │ 施工过程管控            "沟通记录、节点跟进、增项预警"

Step 4 │ 守住底线              用 AI 做风险审查
       │ 合同与交付风险          "合同对比、条款审查、付款节点监控"

Step 5 │ 用好工具              把 AI 变成日常判断的延伸
       │ AI工具 + AI知识库      "不是 AI 替你做判断，
                               是你用 AI 把判断做得更准"
```

**培训时的翻页节奏：** 左边讲一个真实装修案例（2-3 分钟）→ 右边抽出一条 AI 方法（2-3 分钟）→ 现场打开 AI 工具演示（1-2 分钟）→ 下一张。

装修同行看左边就够了。其他行业的人看右边就够了。两个受众群体，一套 PPT。

## 六、各版块内容设计

### 1. 装修美学 —— 案例根基 + 信任基础设施

**定位：** 不是商业产品，是整个故事的证据链。培训时不展开讲，但台下的人翻到这里会自动信任你。

**80% 的内容投入在这里。** 网站靠它吃搜索流量，培训靠它建立信任。

**2.1 空间方案与审美（最软的入口）**

| 维度 | 内容 |
|---|---|
| 核心问题 | "装修不是把房子装完，而是把你想要的生活落地" |
| 内容类型 | 生活方式判断、空间秩序分析、审美框架、真实案例拆解 |
| 内容形式 | 长篇判断文章（2000-4000字）+ 照片/平面图标注 |
| 更新频率 | 每两周 1 篇 |
| SEO关键词 | 装修风格怎么选、户型改造思路、家庭收纳规划、居住需求梳理 |

**2.2 材料与工艺判断（搜索引擎主要猎物）**

| 维度 | 内容 |
|---|---|
| 核心问题 | "商家说的和实际用的，差在哪三道工序上" |
| 内容类型 | 材料横评、工艺标准对照、品牌真相、参数解读 |
| 内容形式 | 中篇技术文 + 对比表格 + "商家话术 vs 实际意思"对照 |
| 更新频率 | 每两周 1 篇 |
| SEO关键词 | 瓷砖怎么选、乳胶漆品牌对比、定制柜板材、地板材质区别 |

**2.3 施工过程管控**

| 维度 | 内容 |
|---|---|
| 核心问题 | "选对人、说对话、盯对点——每个角色都关键" |
| 内容类型 | 选人判断、沟通方法、工地实录、增项识别、节点验收 |
| 内容形式 | 实操指南 + 检查清单 + 对话还原 |
| 更新频率 | 每月 1-2 篇 |
| SEO关键词 | 装修工人怎么选、工地怎么看、装修增项怎么防、验收清单 |

**2.4 合同与交付风险（离钱最近）**

| 维度 | 内容 |
|---|---|
| 核心问题 | "签字之前，你要看清楚什么" |
| 内容类型 | 报价单拆解、合同条款分析、付款节点判断、交付标准 |
| 内容形式 | 真实报价单标注图 + 条款逐行解读 + 风险信号清单 |
| 更新频率 | 每月 1 篇 |
| SEO关键词 | 装修合同陷阱、报价单怎么看、装修付款节点、预算超支怎么办 |

**2.5 床垫选购（差异化壁垒）**

| 维度 | 内容 |
|---|---|
| 核心问题 | "从材料到合同，把床垫买明白" |
| 内容形式 | 「行业老兵说真话」风格，不做百科式罗列 |
| 更新频率 | 每月 1 篇 |
| 分发重点 | 小红书（床垫选购是流量品类） |

---

### 2. AI知识库 —— 核心商业内容 + 培训主讲材料

**定位：** 不是 AI 教程站，是「一个传统行业的人怎么用 AI 重新做判断」的过程档案。**这是培训时翻得最多的章节。**

| 维度 | 内容 |
|---|---|
| 核心问题 | "AI 能帮我做什么判断？不能替我做什么判断？" |
| 内容类型 | AI工作流拆解、提示词库、工具实测、AI辅助判断的边界 |
| 内容形式 | 教程 + 截图 + 实际输出对照 |
| 更新频率 | 每月 1-2 篇 |
| 关键原则 | 每篇落在具体场景。不写"AI 的 10 种用法"，写"我把 3 份报价单扔给 AI，它帮我找到 7 个漏项" |
| 培训用途 | 模块 A/B 的内容直接来自这里。写完一篇文章 = 培训多了一页 PPT |
| SEO关键词 | AI装修设计、AI辅助决策、传统行业AI转型、AI提示词模板 |

---

### 3. AI工具 —— 培训时的"活证据" + 商业转化引擎

**定位：** 培训时最有力的时刻——不是讲 PPT，是现场打开工具演示。台下的人看到 AI 真的能干这个，比讲 10 分钟理论有用。

| 工具 | 培训场景 | 转化路径 |
|---|---|---|
| 报价风险初筛 | "现在我把这份真实报价单扔进去" | → 人工快审服务 |
| AI 居住诊断 | "假设你是一个业主，点这个" | → 咨询服务 |
| 预算结构拆解 | "看看你的预算有没有漏项" | → 预算咨询 |
| 施工验收清单 | "从第 1 天盯到验收的 12 个节点" | → 节点判断服务 |

**每个工具页标配：**
- 工具本身（交互）
- 一行"谁适合用"（3 秒判断）
- 一行"这个工具不能替你做什么"（建立信任，也是培训时必讲的一句话）
- 一篇配套解说文章（放在 AI知识库或装修美学对应子项下）

---

### 4. IP孵化 —— 给同行看的成长档案

**定位：** 不是教别人做 IP，是「我自己正在做，这是过程记录」。培训模块 A 里的"一人公司模式"章节直接引用这里的内容。

| 维度 | 内容 |
|---|---|
| 核心问题 | "一个传统行业的人，怎么用内容把自己重新做一遍？" |
| 内容类型 | 内容创作方法、个人品牌定位、信任建立、数据复盘 |
| 更新频率 | 每月 1 篇 |
| 与其他版块 | IP孵化 → OPC社群（理论 → 实践）；IP孵化 → AI知识库（IP方法 → AI工具） |

---

### 5. OPC·同行有你 —— 商业闭环

**定位：** 社群是培训的自然延伸——听完课的人想继续跟，就进社群。

| 子项 | 内容 |
|---|---|
| 一人公司方法论 | 模式设计、收入结构、成本控制 |
| 实战项目 | 真实项目复盘（脱敏） |
| 社群动态 | 活动记录、成员故事 |
| AI成长营 | 轻培训、新手转型 |

**商业漏斗：** 网站免费内容 → 培训 → 付费社群 → 一对一辅导

更新频率：每月 1-2 篇公开复盘。

---

### 6. 关于我 —— 信任锚点 + 培训客户入口

**已有亮点：** 判断年表（保留，全站最有记忆点的内容）

**需要补：**
1. 一张真实的照片
2. 「邀请我分享/培训」板块——培训三个模块 + 联系方式
3. 从关于页到培训主题的清晰出口

## 七、培训场景设计

### 培训三个模块（修正版）

```
模块 A｜传统行业经营者怎么用 AI 把自己重做一遍 ★ 主打产品
  - 一个装修老兵的真实过程：从"不会用"到"离不开"
  - 核心方法：把经验结构化、用 AI 做对比决策、风险管理
  - 每步配一个装修真实案例 + 一个通用方法论
  - 适合：商会、行业协会、创业社群、中小企业主
  - 时长：2-3 小时
  - 网站对应：AI知识库 + AI工具（现场演示）

模块 B｜AI 工具实操：从"不敢用"到"离不开" ★ 主打产品
  - 全程屏幕录制级实操，不讲概念
  - 真实演示：报价单×AI、合同审查×AI、信息整理×AI
  - 学员跟着操作，带走自己的第一个 AI 工作流
  - 适合：企业内训、团队技能提升、社团活动
  - 时长：1.5-2 小时
  - 网站对应：AI工具（现场演示）+ AI知识库（课后资料）

模块 C｜装修判断方法论 ★ 仅面向同行/装修相关行业
  - 16年经验的系统化输出
  - 报价、合同、施工管控、材料工艺
  - 适合：装修公司内训、设计师社群、物业工程部门
  - 时长：2-3 小时
  - 频次：不是主力产品，有人请才讲
  - 网站对应：装修美学（案例库）
```

### 网站即 PPT 的使用方式

培训时不需要单独做 PPT。直接打开网站，按导航顺序翻：

```
1. 打开首页        → 自我介绍，核心观点（3 分钟）
2. 翻到装修美学     → "先给你们看几个真实案例"（选 1-2 个）（5 分钟）
3. 翻到 AI知识库    → "这是我怎么用 AI 的"（核心部分，20-30 分钟）
4. 打开 AI工具      → 现场演示（最有力的环节，10-15 分钟）
5. 翻到 IP孵化      → "我的成长过程"（5 分钟）
6. 翻到 OPC社群     → "不只我一个人，有一群人在干"（3 分钟）
7. 停在 关于我      → "想请我来讲，联系方式在这里"（2 分钟）
```

每个页面设计时都要考虑投影效果：字号够大、对比度够高、翻页有节奏。

### 培训案例页设计标准

```
[封面]  实拍照片 + 一句话判断（36号字，最后一排能看清）

[问题]  "业主说：XXXX" / "看起来没问题，但其实——"

[拆解]  逐条标注（适合翻页，每条一屏）
        - 哪一行有问题
        - 问题本质是什么
        - 应该怎么改

[对照]  调整前 vs 调整后（表格）

[提炼]  案例背后的通用方法论（1-2 条，任何行业都能用）
```

### MVP 培训案例（先做 3 个）

1. 「一份 28 万的报价单，7 个漏项」—— 装修案例，对应 AI 方法论"用 AI 做信息对比"
2. 「水电暂估 1.2 万变 3.7 万」—— 装修案例，对应 AI 方法论"用 AI 做风险识别"
3. 「我把报价扔给 AI，它找到了人没发现的」—— AI 案例，培训模块 A/B 通用

## 十、内容生产优先级

### 第一轮：建根基 + 备培训（第 1-2 个月）

| 优先级 | 做什么 | 理由 |
|---|---|---|
| P0 | AI知识库 2 篇（报价×AI、传统行业×AI方法） | 培训模块 A/B 的核心材料 |
| P0 | 3 个 MVP 培训案例页 | 上了讲台得有东西翻 |
| P0 | 合同风险 2 篇 | SEO 金矿 + 培训案例素材 |
| P1 | 空间方案 2 篇 | 案例根基 |
