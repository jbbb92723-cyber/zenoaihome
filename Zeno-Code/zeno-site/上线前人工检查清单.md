# 上线前人工检查清单
**版本**：v1.0 | 2026-04-19  
**用法**：部署 Vercel 之前，逐条核对。打开本地 `npm run dev` 或部署后的链接，对着清单手动检查一遍。

---

## 第一轮：内容与表达检查

### 1. 首页 — 一眼知道你是谁

- [ ] 打开首页，5 秒内能判断「这个人是做什么的」
- [ ] Hero 标题是「从装修出发，聊居住、美学、人性、成长与 AI 时代的长期主义」，表达清晰
- [ ] 信任卡片里的身份描述真实（16 年装修从业经历 / 从工地看世界 / AI 升级实践者）
- [ ] 内容方向 5 个板块文字准确
- [ ] 资料库入口 3 条预览文案不含虚假描述
- [ ] 底部「关于我」引导语自然，不是硬推销

### 2. 关于页 /about — 真实可信

- [ ] 「我是谁」段落真实，有具体经历（工地、施工、方案、团队）
- [ ] 「我相信什么」4 条是你自己认可的立场
- [ ] 「我不做什么」3 条边界清晰
- [ ] 「我现在在做什么」4 条是真实现状
- [ ] 联系方式入口存在（微信号 / 邮箱 / 公众号均已填写真实值）
- [ ] 没有「保证」「承诺」「最快」之类的营销腔

### 3. 文章列表页 /blog — 整体印象

- [ ] 6 篇文章全部可见
- [ ] 分类筛选器正常（点「居住与装修」只看装修文章，点「全部」显示全部）
- [ ] 每篇文章的摘要 1-2 行，不截断中文
- [ ] 点击任意文章能正确跳转

### 4. 文章详情页 /blog/[slug] — 适合阅读

- [ ] 文章标题、分类、发布日期正确显示
- [ ] 正文段落行距舒服，中文阅读无压迫感
- [ ] 作者区有「Zeno · 赞诺 / 16 年装修从业经历」
- [ ] 结尾有「去资料库看看」CTA
- [ ] 结尾有相关文章推荐（至少 1 篇）
- [ ] 「装修预算为什么总超？」文章末尾的 FAQ 区块显示正常

### 5. 专题页 /topics — 像长期栏目

- [ ] 5 个专题全部显示，不缺不漏
- [ ] 每个专题有核心问题、描述、会写什么
- [ ] 「已有文章」链接能正常点击跳转
- [ ] 建议阅读顺序提示清晰

### 6. 资料库 /resources — 不像下载站

- [ ] 5 份资料全部显示
- [ ] 每份资料有「适合谁」「解决什么问题」「怎么用」「注意事项」
- [ ] 领取方式统一为「关注公众号 + 回复关键词」
- [ ] 底部有「回去看文章」和「看专题」入口
- [ ] 没有「限时免费」「仅限今天」之类的虚假紧迫感

### 7. 服务页 /services — 看得懂、能联系

- [ ] 4 项服务全部显示
- [ ] 每项服务有价格（非【待补充】）
- [ ] 每项服务有「适合谁」和「不适合谁」（诚实的边界说明）
- [ ] 联系方式：微信号和备注说明清晰
- [ ] 「服务不接所有咨询」的说明存在
- [ ] 底部「查看联系方式」CTA 能点到 /contact

### 8. 联系页 /contact — 有入口

- [ ] 3 种联系方式（公众号 / 微信 / 邮箱）全部显示真实值
- [ ] 「适合联系我的情况」5 条清晰
- [ ] 「不太适合的情况」4 条存在
- [ ] 「建议这样发消息」提示清晰
- [ ] 没有「立刻回复」「24 小时在线」之类的虚假承诺

---

## 第二轮：技术与视觉检查

### 9. 导航与路由

- [ ] Header 导航 7 个链接全部可点（首页 / 关于我 / 文章 / 专题 / 资料库 / 服务 / 联系）
- [ ] 当前页面对应导航项高亮（active 状态）
- [ ] 点「Zeno」logo 回到首页
- [ ] Footer 导航同样可点
- [ ] Footer 联系方式与联系页一致

### 10. 手机端适配

- [ ] 用手机（或浏览器手机模式）打开首页，布局正常，不横向溢出
- [ ] 手机端 Header 有汉堡菜单，能展开关闭
- [ ] 手机端文章正文字号舒适，可以正常阅读
- [ ] 手机端各页面底部不被截断

### 11. 图片与媒体

- [ ] 没有破损图片图标出现（如果没有真实图片，显示字母占位「Z」）
- [ ] 头像区域显示「Z」字母（或真实头像）
- [ ] 文章封面区域：无封面时不显示占位块，只显示文字卡片
- [ ] 页面没有因图片导致的横向滚动

### 12. 样式一致性

- [ ] 所有内页（/about /blog /topics /resources /services /contact）都有 PageHero 区块
- [ ] 所有页面 Header / Footer 样式一致
- [ ] 按钮样式统一（主按钮深褐色 / 次按钮描边 / Ghost 文字下划线）
- [ ] 卡片边框颜色、hover 效果统一

### 13. 可见内容检查

- [ ] 全站搜索「【待补充】」，页面上不应出现（代码层面可以，对外可见页面不行）
- [ ] 没有「TODO」「FIXME」「测试内容」等开发遗留文字
- [ ] 没有编造的电话号码、地址、客户评价

---

## 第三轮：SEO 基础检查

### 14. Meta 信息

- [ ] 首页 title 包含「Zeno」和装修关键词
- [ ] 各内页 title 格式为「页面名称 · Zeno」
- [ ] 文章页有描述性 meta description
- [ ] 文章页有 OpenGraph 数据（在微信分享卡片里预览一下）

### 15. 技术文件

- [ ] 访问 `/robots.txt` 能看到正确内容（允许所有爬虫）
- [ ] 访问 `/sitemap.xml` 能看到所有页面的 URL
- [ ] 访问 `/llms.txt` 能看到网站说明

---

## 第四轮：最终确认

- [ ] 本地 `npm run build` 通过，无报错
- [ ] Vercel 部署成功（状态为绿色）
- [ ] 用手机扫码访问部署地址，确认正常
- [ ] 公众号关键词自动回复已设置（资料领取路径通畅）
- [ ] 告诉 1-2 个真实用户，请他们试用并反馈
