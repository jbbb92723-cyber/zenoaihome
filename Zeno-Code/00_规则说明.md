# Zeno-Code 规则说明

## 定位

这里存放 ZenoAIHome 网站前端、后台、接口、数据库模型和整套程序源码。

## 可以放这里

- 正式官网项目源码，例如 Next.js `app/`、`components/`、`lib/`、`data/`、`styles/`。
- 后台管理页面、API 路由、鉴权、订单、资源领取、内容发布接口。
- 数据库 schema、迁移脚本、服务端校验、业务逻辑代码。
- 程序运行必需的配置文件，例如 `package.json`、`tsconfig.json`、`next.config.*`、Tailwind、ESLint 配置。
- 与代码强相关的 README、开发说明、部署说明。

## 不放这里

- 内容资产库、文案草稿、素材原图、发布稿。
- 纯运营文档、研究笔记、个人知识库。
- 本地生成物，例如 `node_modules`、`.next`、`out`、缓存、日志。
- 不确定是否仍使用的旧版本源码、测试版、半成品。

## 归类原则

如果一个文件主要回答“网站/后台如何运行、如何构建、如何部署”，优先归入这里。

如果是旧源码、备份包、测试项目或半成品，不直接放入正式代码目录，先放入 `待归类文件夹` 等待确认。
