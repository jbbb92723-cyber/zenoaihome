## 二、日常内容发布 SOP

### 判断笔记（核心日常）

判断笔记是你的轻武器。比文章轻、比朋友圈重。写笔记是你站点最频繁的操作。

**判断笔记 ≠ 博客文章**
- 笔记：300-800 字，一个判断瞬间，不需要 intro/结语/配图
- 文章：1500+ 字，结构化长文，需要 SEO metadata

**发布流程（每次 5-10 分钟）**：

```
[ ] 1. 打开 /admin/notes，点「+ 新建笔记」
[ ] 2. 标题：一句话说清这篇在判断什么
        ✅ 「我看报价单只看三行」
        ✅ 「今天一个业主问了水电上限，我给她看了三张图」
        ❌ 「装修报价避坑指南」——太像文章了
[ ] 3. 正文：直接开始写。不要 intro。不要"大家好"。不要 CTA。
        第一句就是具体的场景——
        "今天又看了一份报价。南宁青秀区。"
[ ] 4. 写完切到「预览」tab，从头读一遍
[ ] 5. 过全息铁律检查表（见下方）
[ ] 6. 分类选好，标签 3-5 个，状态切「公开」
[ ] 7. 点「创建并发布」
[ ] 8. 打开 /notes 看一眼前台效果
```

### 全息铁律检查表（发布前必过）

```
[ ] 这篇里有一个具体的真实瞬间吗？
    时间/地点/金额/人物/后果 —— 至少一个。
    ❌ "水电容易超预算"
    ✅ "南宁青秀区，暂估 2.3 万，结算 11.7 万"

[ ] 小白读者能拿走一个具体动作吗？
    读完知道在报价单第几行找哪三个字。
    ✅ "打开你的报价单，找水电那行，看有没有'上限'两个字"

[ ] 同行能看出判断结构吗？
    不是"他懂装修"，是"他的判断路径我理解了"
    ✅ "这不是诚意问题，是结构问题——暂估把不确定性全甩给了业主"

[ ] AI 创作者能闻到水泥灰吗？
    有没有一句话，AI 不可能自己写出来？
    ✅ "我蹲在工地地上用手电筒照了半天才确认"

[ ] AI Agent 能抓到至少 3 个事实锚点吗？
    南宁 / 青秀 / 2.5 万 / 按实结算 / 水电 / 暂估 / 首付 60% ...
    → 这些词是你的站被 AI 推荐的锚

[ ] 给 Claude 同样的提示词，它能生成等价内容吗？
    能 → 这篇是全信息，不是全息内容。重写。
    不能 → 发。
```

### 更新首页签名（可选，每周一次）

首页 Hero 下方有一行小字："今天在看的：XXX"。建议每周或每两周换一次，反映你当下最关注的问题。

编辑文件：`components/features/home/HomePageGptSkill.tsx`
找到 `今天在看的：` 那行 → 改掉 → commit → push。

### 发布文章（低频，每月 1-4 篇）

文章数据在 `data/content/articles.ts` 和 `data/content/article-XX-content.ts` 中。修改流程：
1. 编辑对应文件
2. 本地 `npx.cmd tsc --noEmit` 确认编译
3. commit → push

## 三、安全注意事项

### 密码管理
- `ADMIN_PASSWORD` 在 Vercel 环境变量中，不要分享
- 如果怀疑泄露：Vercel Dashboard → Settings → Environment Variables → 改掉 → Redeploy

### Cookie 安全
- 管理员 session 7 天有效，过期需重新登录
- Cookie 设置：httpOnly ✅ · secure（生产）✅ · sameSite lax ✅

### 操作回溯
- 所有管理操作自动记入 `/admin/logs`
- 怀疑异常时，先去日志页看最近操作
- 日志保留最近 100 条

## 六、常见问题

**Q: 登录过期了怎么办？**
A: 浏览器打开 `/admin/login`，重新输入 `ADMIN_PASSWORD`。有效期 7 天。

**Q: 笔记发了但前台看不到？**
A: 检查笔记状态是不是「公开」（PUBLIC）。草稿和私有笔记不会出现在前台 `/notes` 页。

**Q: slug 冲突了？**
A: 系统会提示"slug 已存在"。改一下 slug 就行，比如后面加个 `-2`。

**Q: 删掉的笔记能恢复吗？**
A: 不能。删除前系统会弹确认框。操作记录会留在日志里但内容不可恢复。
