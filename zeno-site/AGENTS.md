# AGENTS.md｜ZenoAIHome 官网代码协作规则

> 本文件适用于 `zeno-site/` 内的所有官网代码协作。进入本目录工作时，优先遵守本文件。

## 1. 官网定位

ZenoAIHome 是「美学 × 生活 × 装修决策 × 风险控制」的居住判断系统。

它帮助业主把理想生活翻译成：

- 可落地的空间方案
- 可判断的装修决策
- 可承受的预算边界
- 可写进合同的约定
- 可提前识别的交付风险

报价风险仍然重要，但它不是全部。它是「理想生活能否被装修真正落地」中的一个风险入口。

## 2. 技术边界

当前项目是 Next.js App Router 官网。

- `app/`：只放路由、页面入口、metadata、页面级编排。
- `components/ui/`：通用 UI 组件。
- `components/layout/`：站点框架、导航、搜索、全局浮窗。
- `components/features/`：按业务域归组的页面模块。
- `data/`：内容、服务、工具、诊断、风险控制等结构化数据。
- `lib/`：纯逻辑、工具函数、服务端辅助函数。
- `public/`：线上实际引用的静态资源。
- `styles/`：全局样式、设计系统和 Tailwind 基础层。

不要绕开现有结构新建平行目录。新增页面或组件前，先搜索是否已有同类入口。

## 3. 信息架构主线

官网表达顺序应保持：

```text
理想生活
  ↓
美学与生活方式
  ↓
空间方案与家庭场景
  ↓
装修决策
  ↓
预算 / 报价 / 合同 / 施工 / 交付风险
  ↓
ZenoAIHome 工具与服务
```

不要把网站做成装修知识大全。不要把网站做成 AI 工具展示站。它首先是一个帮助业主做装修重大决策的判断系统。

## 4. UIUX 规则

当前视觉方向：`Zeno Monastic Editorial`。

三主色：

- Bone Paper：`#F4F1E8`
- Carbon Ink：`#111111`
- Warm Sand：`#DED2BE`

允许使用透明度、中性色、阴影和边框层级，但不得形成第四个主色。

设计原则：

- 中文标题要有高级、克制、编辑感。
- 组件边角保持克制，默认不超过 `8px`。
- 页面不走普通 SaaS 卡片风。
- 页面不走装修公司模板风。
- 页面不堆大段说明文案。
- 动效是「判断仪器式」，不是炫技动画。
- 移动端必须成立，文字不能溢出或互相遮挡。

首屏应该有明确视觉锚点，例如：

- 居住判断矩阵
- 空间秩序雷达
- 生活到交付判断链
- 风险扫描仪表

## 5. 文案规则

语气应该是：

- 温和
- 专业
- 有判断
- 不制造焦虑
- 不油腻
- 不模板化
- 能把复杂问题说清楚

禁止表达：

- 千万别
- 装修小白必看
- 避坑秘籍
- 被坑惨了
- 99% 的人不知道
- 一招教你
- 高端生活方式
- 松弛感拉满
- 梦中情房
- AI 赋能装修新时代

推荐表达：

- 这不一定有问题，但签约前要问清楚。
- 好看的设计，必须经得起日常生活。
- 报价不是单独看的，它应该能承接你的空间选择和生活目标。
- ZenoAIHome 不帮你制造焦虑，只帮你把看不清的地方拆开。

## 6. 服务边界

所有服务文案必须保留边界：

- 不代替业主签约。
- 不承诺最低价。
- 不制造业主与装修公司的对立。
- 不替代法律审查。
- 重点是帮助业主在签约前看清风险和取舍。

## 7. 修改流程

### 可以直接执行

- 单页文案精修。
- 单组件视觉细节。
- 明确 bug 修复。
- 明确数据项修正。

### 必须先审计

- 首页、服务页、导航、信息架构调整。
- `components/`、`data/`、`app/` 的目录重组。
- 删除旧页面、旧组件、旧数据。
- 新增后台模型、订单流程、支付流程。
- 引入新依赖。

## 8. 验证命令

修改 TypeScript、React、数据或样式后，优先运行：

```bash
npx.cmd tsc --noEmit
npm.cmd run lint
npx.cmd next build
```

如需运行完整 build：

```bash
npm.cmd run build
```

如果 `npm run build` 卡在 `prisma generate` 替换 `node_modules/.prisma` Windows DLL 文件，先检查是否是本机 Node 进程占用。不要把这个本地权限问题误判为页面构建失败。

## 9. 提交规则

- 只提交与当前任务相关的 `zeno-site/` 文件。
- 默认不提交 `.next/`、`node_modules/`、`tsconfig.tsbuildinfo`。
- 默认不提交未被线上引用的 `public/design-previews/`。
- 不混入内容资产库删除记录和本地 crash log。
- 用户明确要求 push 时，提交后推送到当前分支远端。
