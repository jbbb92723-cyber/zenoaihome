# 微信公众号文章：OpenClaw + Obsidian 内容工厂

## 原文核心

这篇文章讲的是：用 Agent + Obsidian 搭建内容工厂。

它的核心结构是：

1. 用 Obsidian 做本地 Markdown 知识库。
2. 用 OpenClaw 做后台执行引擎。
3. 把外部内容抓取、灵感沉淀、选题生成、大纲、初稿、发布归档做成流程。
4. 用 YAML 元数据标记每篇内容所处阶段。
5. 用 SOP、USER、SOUL、AGENTS 等规则文件约束 Agent 行为。
6. 用多 Agent 分工：研究员、小红书、公众号、总管。

## 可以吸收的营养

对 ZenoAIHome 有用的不是“爆文工厂”这个说法，而是它背后的系统思想：

> 内容不是一次性生成，而是从真实经验、外部素材、用户问题、平台反馈中不断沉淀成资产。

可以吸收为 ZenoAIHome 的工作流：

- 外部素材进入 `01_灵感收集箱`
- 装修案例、报价截图、用户评论进入 `03_素材证据库`
- 可复用判断方法进入 `03_素材证据库/方法卡`
- 可发布内容进入 `04_内容资产单元`
- 服务说明和咨询脚本进入 `05_产品转化/服务`
- Agent 工作流和提示词进入 `06_提示词库`
- 审核后的公开内容进入 `09_网站母站`

## 不要照搬的部分

1. 不要追“100W+爆文”“单篇6000元”这种钩子。
2. 不要把 ZenoAIHome 做成泛内容工厂。
3. 不要为了产量牺牲真实装修判断。
4. 不要让 AI 直接复制爆款结构，变成无人格的流水线内容。
5. 不要把小红书、公众号、网站都当成同一种内容出口。

ZenoAIHome 的核心不是“爆文”，而是：

> 把 16 年装修现场经验，系统化成业主签约前能用的判断工具。

## 对 ZenoAIHome 的翻译

原文模型：

> 爆款素材库 → 选题池 → 内容工厂 → 发布归档

ZenoAIHome 应该改成：

> 真实问题库 → 风险标签库 → 判断方法库 → 内容/工具/服务交付

更适合 ZenoAIHome 的内容资产闭环：

1. **收集**：业主问题、报价单风险词、工地观察、同行话术、投诉案例。
2. **拆解**：识别报价风险、合同风险、施工交付风险、业主误判。
3. **沉淀**：形成风险标签、检查清单、咨询问题、方法卡。
4. **生产**：变成小红书图文、公众号文章、网站 SEO/GEO 文章。
5. **转化**：进入报价审核服务、合同联审服务、验收清单服务。
6. **反馈**：把用户咨询、收藏、评论、成交问题反哺知识库。

## 可转化动作

- 做一张方法卡：`ZenoAIHome内容资产飞轮：从装修经验到报价审核服务`
- 更新服务流程：报价审核不只是人工看单，而是“素材库 + 风险标签 + AI 初筛 + 人工判断”的系统。
- 给 Hermes/Obsidian 建立更清楚的生产规则：不同素材进入不同目录，不再散落。
- 小红书继续做风险词系列，但每篇都回流到风险标签库。
- 公众号不要追爆文，要写“为什么这个风险值得签约前看懂”。

## 一句话结论

这篇文章证明了：Agent + Obsidian 的价值不在于多写几篇文章，而在于把人的经验变成可复用、可检索、可交付、可迭代的资产系统。

ZenoAIHome 要赢它，不是比谁更会做爆文，而是比谁更有真实行业判断、更垂直的风险标签、更强的服务闭环。
