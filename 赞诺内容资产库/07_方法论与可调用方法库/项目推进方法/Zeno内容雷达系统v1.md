# Zeno 内容雷达系统 v1

> 这不是“爆款监控系统”，而是 Zeno OS 用来持续捕捉外部内容信号、用户痛点、报价风险词和可转化选题的内容资产雷达。

## 一句话定义

**Zeno 内容雷达系统 = 外部信号监控 + 增量数据记录 + 每日战报 + AI 拆解 + 内容/服务资产沉淀。**

它服务于 [[../Zeno判断资产系统总图|Zeno 判断资产系统总图]] 的输入层、理解层、资产层和回填层。

扩展设计稿：[[Zeno内容中台搭建自动化与变现设计稿|Zeno 内容中台搭建：自动化与变现设计稿]]。

## 内容中台定位

内容雷达只是输入系统；Zeno 真正要搭的是内容中台。

```text
内容中台 = 内容雷达 + 判断资产库 + 人工写作支持 + 服务转化系统
```

它的终点不是“报告更漂亮”，而是：

- 找到真实需求。
- 形成 Zeno 自己的判断资产。
- 支持人工写作和服务交付。
- 缩短从内容到付费咨询/报价审核/AI 工作流教学的路径。

## 不做什么

- 不以“追爆款”为唯一目标。
- 不照抄博主内容、结构、案例和原文。
- 不自动生成最终逐字稿。
- 不一开始追求 100+ 账号、全平台、全自动。
- 不把所有监控内容都塞进 ZenoAIHome。

## 做什么

- 找到真实用户正在讨论的问题。
- 识别装修报价、签约前判断、传统行业 AI、内容资产化等高价值信号。
- 把外部内容拆成痛点、机制、选题、服务问题、方法卡。
- 每天/每周给 Zeno 推送少量值得看的内容。
- 把高价值信号沉淀进 Zeno OS，而不是停留在收藏夹。

## v1 最小闭环

```text
10-30 个对标来源
→ 凌晨/固定时间采集新增内容
→ 去重、清洗、增量计算、历史快照
→ 爆款/高价值指数初筛
→ 早上推送 3-5 条内容雷达战报
→ Zeno / 运营手动标记待拆解
→ AI 拆解成痛点卡 / 选题卡 / 服务问题 / 方法卡候选
→ 写入 Obsidian / 飞书 / 云文档
→ 发布或服务后复盘回填
```

## 推荐工具分工

### Hermes

- 定时任务
- 网页/资料抓取
- 文本清洗
- AI 拆解
- Obsidian 写入
- 微信/飞书推送

### 飞书多维表

- 作为可视化数据表和轻量看板。
- 适合来源库、内容库、选题库、战报视图。

### Obsidian

- 作为长期知识资产真源。
- 保存方法卡、痛点卡、旧观点卡、素材膨胀记录、人工写稿素材包。

### gstack

- 用于检查 zenoaihome.com 的页面、表单、CTA、内容转化路径。
- 不负责内容采集本身。

### Coze

- 可作为低代码抓取/调度层。
- 但不是必须；如果 Hermes 能做，优先用 Hermes 保持本地资产可控。

## 五个核心库

### 1. 来源库

用途：记录监控谁、为什么监控、是否继续监控。

建议字段：

| 字段 | 说明 |
|---|---|
| source_id | 来源唯一 ID |
| 平台 | 抖音 / 小红书 / 公众号 / B站 / 视频号 / 网站 / 社群等 |
| 来源名称 | 账号名、网站名、群名或资料源名称 |
| 主页链接 | 来源主页或固定入口 |
| 来源类型 | 对标博主 / 竞品 / 客户声音 / 行业资料 / 个人旧素材 / 其他 |
| 归属项目 | Zeno OS / ZenoAIHome / OPC / 暂存 |
| 赛道标签 | 装修报价、签约、AI、传统行业、内容方法等 |
| 监控价值 | 高 / 中 / 低 |
| 监控频率 | 每日 / 每周 / 手动 |
| 是否启用 | 是 / 否 |
| 最近采集时间 | 时间戳 |
| 备注 | 为什么值得看 |

### 2. 内容库

用途：记录每条被采集到的视频、文章、帖子、评论串。

建议字段：

| 字段 | 说明 |
|---|---|
| content_id | 内容唯一 ID |
| 标题 | 原始标题或自动摘要标题 |
| 链接 | 原文链接 |
| 平台 | 内容所在平台 |
| 来源名称 | 关联来源库 |
| 发布时间 | 原始发布时间 |
| 采集时间 | 系统采集时间 |
| 内容类型 | 视频 / 图文 / 文章 / 评论 / 直播切片 / 网页 |
| 原始文本 | 摘要或正文文本 |
| 数据指标 | 点赞、收藏、评论、转发、阅读等 |
| 增量指标 | 新增点赞、新增收藏、新增评论等，尽量精确到个位数 |
| 历史快照 | 上次抓取数据、当前抓取数据、变化区间 |
| 高价值指数 | 综合收藏率、评论需求、增量速度、Zeno 相关度 |
| 热度等级 | 高 / 中 / 低 |
| 人工标记 | 待拆解 / 不拆 / 重点跟踪 / 已处理 |
| 状态 | 待处理 / 已拆解 / 已入库 / 已转化 / 丢弃 |
| 云文档/Obsidian路径 | 拆解文档或资产沉淀位置 |
| 初筛结论 | 为什么保留或丢弃 |

### 3. 痛点库

用途：记录真正有价值的用户表达和需求信号。

建议字段：

| 字段 | 说明 |
|---|---|
| pain_id | 痛点唯一 ID |
| 用户原话 | 评论、标题、私信、咨询里的原始表达 |
| 背后问题 | 用户真正不会判断什么 |
| 决策场景 | 签约前 / 比报价 / 看合同 / 施工中 / 验收 / AI 学习 / 内容创业 |
| 归属项目 | ZenoAIHome / OPC / Zeno OS |
| 风险词 | 如按实结算、暂估、另计、不含、品牌材料等 |
| 服务追问 | 面对这个痛点，服务时该问什么 |
| 内容角度 | 可写成什么内容 |
| 资产价值 | 高 / 中 / 低 |
| 后续动作 | 入选题库 / 入服务 SOP / 入方法卡 / 暂存 |

### 4. 选题库

用途：把内容信号变成可执行选题，而不是收藏夹。

建议字段：

| 字段 | 说明 |
|---|---|
| topic_id | 选题唯一 ID |
| 选题标题 | Zeno 自己的标题，不照抄原题 |
| 来源内容 | 关联内容库 |
| 目标用户 | 业主 / 传统行业老板 / AI 学习者 / 个人 IP 关注者 |
| 决策场景 | 用户在哪个具体场景会需要它 |
| 核心误判 | 这条内容要纠正什么误判 |
| 适合平台 | 网站 / 公众号 / 小红书 / 短视频 / 朋友圈 |
| 可转化服务 | 报价审核 / 签约前咨询 / AI 培训 / 暂无 |
| 优先级 | A / B / C / 丢弃 |
| 状态 | 待素材膨胀 / 已出素材包 / 已写 / 已发布 / 已复盘 |
| 复盘结果 | 数据和用户反馈 |

### 5. 拆解文档库

用途：保存 AI 拆解后的结构化资产。

建议字段：

| 字段 | 说明 |
|---|---|
| doc_id | 拆解文档 ID |
| 拆解对象 | 关联内容库或选题库 |
| 爆发原因 | 为什么它可能被传播 |
| 开头钩子 | 它如何抓住注意力 |
| 内容结构 | 起承转合、冲突、案例、结论 |
| 用户情绪 | 焦虑、确认、共鸣、反驳、收藏等 |
| 可迁移机制 | 不照抄内容，只提取机制 |
| Zeno 迁移角度 | 迁移到 ZenoAIHome / OPC / Zeno OS 的方式 |
| 可生成资产 | 痛点卡 / 选题卡 / 方法卡 / 服务问题 / 素材包 |
| 审核状态 | AI 初稿 / Zeno 已看 / 已采用 / 丢弃 |

## 爆款/高价值识别规则

## 高价值指数 v1

不要只看点赞数。Zeno 内容雷达的高价值指数建议由 5 个部分组成：

```text
高价值指数
= 增量速度
+ 收藏/评论质量
+ 用户真实需求强度
+ Zeno 业务相关度
+ 可转化资产价值
```

### 参考评分

| 维度 | 说明 | 分值 |
|---|---|---:|
| 增量速度 | 短时间新增点赞、收藏、评论明显高于平时 | 0-20 |
| 收藏/评论质量 | 收藏高、评论具体、有人讲自己情况 | 0-20 |
| 真实需求强度 | 是否暴露明确痛点、决策困难或付费可能 | 0-20 |
| Zeno 相关度 | 是否贴近报价风险、签约前判断、经验资产化 | 0-20 |
| 可转化资产价值 | 能否转成选题、服务追问、方法卡、SOP | 0-20 |

建议阈值：

- 80 分以上：必须拆解。
- 60-79 分：进入选题池或痛点库。
- 40-59 分：暂存，等同类信号重复出现。
- 40 分以下：丢弃。

### ZenoAIHome 优先信号

- 评论区出现“报价看不懂”“怕增项”“合同没写清”“预算超了”等真实表达。
- 内容指向签约前判断、报价边界、责任归属、材料型号、付款节点。
- 收藏率高，说明用户想留着用。
- 低粉账号内容爆发，说明选题本身强。
- 评论里有人发自己的情况，说明有服务机会。

### OPC / AI 培训优先信号

- 传统行业老板对 AI 有兴趣但不知道怎么落地。
- 内容不是炫工具，而是把经验变成流程。
- 讨论一人公司、效率、内容资产、方法库、本地系统。
- 能转成课程小节、咨询脚本、案例拆解。

### 丢弃信号

- 只有情绪，没有具体决策场景。
- 只有鸡汤，没有可迁移机制。
- 只有标题党，评论区没有真实需求。
- 与 Zeno OS / ZenoAIHome / OPC 都无关。
- 需要大量伪造案例才能写。

## 异常处理 checklist

内容雷达只要接自动采集，就必须有熔断和异常记录，避免错误数据污染资产库。

### 抓取异常

- API 返回失败或空数据。
- 单个来源连续失败 3 次。
- 平台限流或需要验证码。
- 页面结构变化导致字段缺失。

处理：暂停该来源，标记为“抓取异常”，不继续写入新数据。

### 数据异常

- 同一 URL / 视频 ID 重复写入。
- 点赞、收藏、评论比上次减少且无合理解释。
- 增量异常大，疑似接口字段错位。
- 标题为空、链接为空、来源为空。

处理：进入异常表或异常备注，不触发战报和 AI 拆解。

### AI 拆解异常

- ASR 转写为空或大量乱码。
- LLM 输出只有泛泛总结，没有痛点、机制、迁移角度。
- 输出包含编造案例或未验证事实。
- 与 Zeno OS / ZenoAIHome / OPC 无关却被误判为高价值。

处理：退回“待人工判断”，不进入方法库。

### 人工熔断

Zeno 可以随时把来源或主题标记为：

```text
暂停监控 / 不再推荐 / 只存不拆 / 需要人工看完再拆
```

## 入库规则

### 高价值

- 进入内容库。
- 生成拆解卡。
- 至少转成一个选题或痛点。
- 必要时进入方法库或服务 SOP。

### 中价值

- 进入内容库或暂存。
- 只做摘要，不深拆。
- 等同类素材出现 3 次后再升级。

### 低价值

- 只记录来源或直接丢弃。
- 不进入方法库。

## 变现路径设计

内容雷达不直接赚钱，赚钱的是它不断喂养出来的服务入口和产品 SOP。

### 路径 A：ZenoAIHome 装修签约前服务

```text
报价/合同/签约风险内容
→ 用户发现自己也有类似不确定性
→ 报价单/合同初筛表
→ 低价诊断
→ 签约前咨询或报价审核服务
```

适合沉淀：报价风险词库、签约前检查清单、服务 intake 问题、网站 FAQ。

### 路径 B：传统行业 AI 工作流教学 / 咨询

```text
Zeno 自己跑通内容中台
→ 形成真实案例
→ 向传统行业老板展示
→ 收费帮他们搭建轻量内容/知识/客户中台
```

这条线可以作为第二业务线，但不能稀释 ZenoAIHome 的主定位。

### 路径 C：Zeno OS 判断资产系统

```text
外部信号 + 用户问题 + 旧观点
→ 方法论与可调用方法库
→ 更快内容生产、更稳服务交付、更高复用率
```

这条线短期不一定直接卖，但它提高 Zeno 的交付效率和长期复利。

## 推荐落地顺序

### 第 1 步：先手动建表

先在飞书多维表或 Obsidian 建 5 个库的字段，不急着自动抓取。

### 第 2 步：先监控 10-30 个来源

包括：

- 装修报价/避坑类账号
- 本地装修/施工类账号
- 传统行业 AI 相关账号
- 一人公司/内容资产类账号
- 你自己的历史内容和旧观点库

### 第 3 步：先做半自动战报

每天或每周由 Hermes 根据新增素材生成战报，推送到微信或飞书。

### 第 4 步：再做自动采集

当字段、筛选规则和拆解模板稳定后，再接 Coze/API/脚本定时采集。

### 第 5 步：接入服务与网站

用 gstack 检查：高潜内容是否能导向网站 CTA、报价初筛、报价审核服务。

## 最小可用版本验收标准

v1 成功不是自动化多炫，而是：

- 每周能稳定发现 10 条以上高价值内容信号。
- 每周至少生成 3 个可写选题。
- 每周至少沉淀 1 张方法卡或服务追问。
- 至少 1 条内容能导向 ZenoAIHome 服务动作。
- 战报不超过 5 分钟能看完。

## 和 Zeno 判断资产系统的对应关系

| 内容雷达环节 | 判断资产系统层级 |
|---|---|
| 来源库 / 内容库 | 输入层 |
| AI 拆解模板 | 理解层 |
| 归属项目判断 | 迁移层 |
| 痛点库 / 选题库 / 拆解文档库 | 资产层 |
| 战报建议动作 / CTA / 服务机会 | 转化层 |
| 复盘结果 / 用户反馈 | 回填层 |
