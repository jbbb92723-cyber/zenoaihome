# 域名部署检查清单

> 写给非程序员的版本。如果你不确定某项，按步骤逐一核对即可。
> 最后更新：2026-04-21

---

## 第一步：先弄清你的 Vercel 地址

每个部署到 Vercel 的项目都有一个 **Vercel 自带的域名**，格式是：
```
https://你的项目名.vercel.app
```

**操作：**
1. 打开 https://vercel.com，登录你的账号
2. 点击 `zeno-site` 项目（或显示你项目名的那个）
3. 看"Domains"区域，找到 `.vercel.app` 结尾的地址

---

## 第二步：用 Vercel 默认域名测试

先在浏览器打开：
```
https://zeno-site.vercel.app/admin/publish
（把 zeno-site 换成你实际的 Vercel 域名）
```

**判断：**
- ✅ 能打开 → 代码正常，问题在域名/DNS，继续第三步
- ❌ 还是 404 → 代码有问题，需要检查部署是否成功

---

## 第三步：确认 zenoaihome.com 指向了哪里

打开这个工具检查 DNS：
👉 https://dnschecker.org/#A/zenoaihome.com

**Vercel 的 IP 是：** `76.76.21.21`（或 Vercel 分配给你的 IP）

如果 DNS 检查结果显示的不是 Vercel 的 IP，而是其他服务器 IP → **域名仍指向旧服务器**。

---

## 第四步：检查 Vercel 项目里的域名绑定

1. 打开 Vercel → 你的项目 → Settings → Domains
2. 看 `zenoaihome.com` 是否在列表里
3. 状态是否显示绿色 ✅（Valid Configuration）

**常见错误状态：**
- `Invalid Configuration` → DNS 没配好
- 列表里没有 `zenoaihome.com` → 还没绑定自定义域名

**如何添加域名（如果没有）：**
1. Vercel → 项目 → Settings → Domains → Add
2. 输入 `zenoaihome.com`，按提示操作
3. Vercel 会给你一个 CNAME 记录或 A 记录，需要去 DNS 服务商添加

---

## 第五步：检查 DNS 服务商设置

你的域名在哪里注册的？（万网/阿里云/Cloudflare/GoDaddy？）

打开那个服务商的控制台，找「DNS 解析」或「域名解析」管理页面。

**Vercel 要求的 DNS 配置（二选一）：**

方式 A（推荐，用 A 记录）：
```
类型：A
主机名：@（或 zenoaihome.com）
值：76.76.21.21
```

方式 B（用 CNAME，适合 www 子域）：
```
类型：CNAME
主机名：www
值：cname.vercel-dns.com
```

---

## 第六步：可能还指向旧 WordPress

**判断方法：**

在浏览器打开 `https://zenoaihome.com/wp-login.php`

- 如果出现 WordPress 登录页 → **域名仍然指向旧 WordPress 服务器**
- 如果出现 404 或跳到你的 Zeno 网站 → 已正确指向 Vercel

**如果仍然指向 WordPress：**
1. 找到你的 WordPress 主机服务商（蓝盾、阿里云 ECS 等）
2. 在 DNS 服务商把 A 记录从旧服务器 IP 改成 `76.76.21.21`
3. DNS 生效通常需要 5 分钟到 24 小时

---

## 第七步：Vercel 项目构建设置检查

进入 Vercel → 项目 → Settings → General

| 项目 | 正确值 |
|---|---|
| Root Directory | `zeno-site`（如果是这个子目录） |
| Framework Preset | Next.js |
| Build Command | `next build`（或留空自动检测） |
| Output Directory | 留空（Next.js 自动处理） |
| Install Command | `npm install` |
| Node.js Version | 18.x 或 20.x |

---

## 综合判断

| 情况 | 原因 | 修复方向 |
|---|---|---|
| Vercel 子域名正常，zenoaihome.com 是 WordPress 404 | DNS 未切换 | 更新 DNS 指向 Vercel |
| Vercel 子域名也是 404 | 代码构建失败 | 查 Vercel 部署日志 |
| zenoaihome.com 能打开首页但 /admin/publish 打不开 | 代码路由问题 | 重新部署项目 |
| 两个域名都正常 | 已修复 | 测试所有页面 |

---

## 出现"这个页面似乎不存在。看起来指向这里的链接有误。也许试着搜索一下？"怎么办？

**这是 WordPress 404 的标准文案**，不是 Next.js。

100% 说明你访问的是旧 WordPress 服务器，域名 DNS 还没切换完成。

请按第三步到第六步操作，把 DNS 从旧服务器改指向 Vercel。
