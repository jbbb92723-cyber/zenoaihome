## 四、Vercel 部署步骤

### 首次部署

1. 登录 [vercel.com](https://vercel.com)
2. 点击 **Add New → Project**
3. 选择 GitHub 仓库 `Zenoaihome.com`（或手动上传）
4. **Root Directory**（根目录）设置为 `zeno-site`
5. Framework Preset 自动识别为 **Next.js**，保持默认
6. 点击 **Deploy**

### 自定义域名

1. 进入 Vercel 项目 → **Settings → Domains**
2. 添加 `zenoaihome.com` 和 `www.zenoaihome.com`
3. 按提示在域名注册商处添加 DNS 记录（A 记录或 CNAME）
4. 等待 DNS 生效（通常 5-30 分钟）

### 后续更新

```bash
# 修改代码后，提交并推送到 GitHub
git add .
git commit -m "update: 描述此次改动"
git push

# Vercel 会自动检测 push 并重新部署
```

## 五、新增文章的操作方法

1. 打开 `zeno-site/data/articles.ts`
2. 在数组末尾添加一个新的文章对象，参考现有文章格式
3. 必填字段：`id`（递增）、`slug`（拼音，无空格，用连字符）、`title`、`excerpt`、`content`、`category`、`tags`、`publishDate`
4. `content` 字段为 Markdown 格式，支持标题 `##`、加粗 `**`、列表 `- `、分隔线 `---`
5. 保存文件，本地用 `npm run dev` 预览，确认无误后提交

## 七、紧急回滚方式

Vercel 每次部署均保留历史版本。若新版本出现问题：
1. 进入 Vercel 项目 → **Deployments**
2. 找到上一个正常版本
3. 点击右侧 **···** → **Promote to Production**
4. 立即回滚，无需重新改代码
