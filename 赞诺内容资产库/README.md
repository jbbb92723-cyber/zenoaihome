# Zeno Content OS｜系统说明书

> 当前系统已经从“ZenoAIHome 内容生产文件夹”升级为 **Zeno OS：个人认知 × 内容表达 × 流量选题 × 商业项目 × Agent 协作系统**。

## 1. 这个系统是什么

它不是博客管理系统，也不是 AI 自动写稿系统。

它的目标是让 Zeno 的输入、观点、方法、项目和商业判断，沉淀成可检索、可调用、可复用的本地资产。

核心原则：

- 不鼓励用所谓 AI workflow 模板化生产内容；
- 重要逐字稿主要由人写；
- AI 负责检索、整理、素材膨胀、反方讨论、结构检查、标题开头辅助；
- 本地可以囤积大量不同方法组件，但不能只有一个套路；
- 网站、公众号、小红书、短视频、课程、咨询，都从同一个资产系统里取材。

## 4. 两个核心项目

### ZenoAIHome

路径：`12_项目库/ZenoAIHome/`

定位：帮助准备装修签约的业主，在签字前看懂报价、合同、增项、付款节点、材料品牌型号和工期交付风险。

### OPC 一人公司 AI 培训

路径：`12_项目库/OPC一人公司AI培训/`

定位：面向传统行业老板、个体经营者、内容创作者，帮助他们把真实经验沉淀成本地资产、方法库、内容材料和服务产品。

## 5. 默认处理流程

新资料进入后：

```text
输入资料
→ 判断价值
→ 判断归属层级
→ 提取观点/机制/方法
→ 必要时做素材膨胀
→ 写入对应目录
→ 记录运行结果
```

对于爆款选题，优先使用：

`14_方法论与可调用方法库/流量选题方法/爆款选题双轨素材膨胀法.md`

## 6. 禁止事项

- 不默认让 AI 写完整最终稿。
- 不把所有资料都往装修项目里塞。
- 不把 Obsidian 当垃圾桶。
- 不删除文件，先归档。
- 不编造案例、数据、价格、客户信息。
- 不用“赋能、抓手、全链路、生态”等空泛术语污染方法卡和读者稿。
