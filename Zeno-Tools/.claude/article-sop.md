# 赞诺文章SOP生产线

> 每次写新文章或改旧文章，按这个流程走。六个环节，每环节有检查点。

---

## ① 定方向（5分钟）

**做什么**：想清楚这篇文章要帮谁、解决什么问题。

**检查点**：
- [ ] 读者看完能少踩什么坑？
- [ ] 这篇文章你16年经验里哪个真实故事能撑住？
- [ ] 一句话说清楚核心观点是什么？

**工具**：不需要skill，自己在笔记里写一句话回答上面三个问题。

**输出**：一句核心观点 + 一个要讲的故事

---

## ② 写初稿（不限时）

**做什么**：把你脑子里的话倒出来。别管AI味，只管真实不真实。

**规则**：
- 像跟朋友微信聊天一样写
- 有故事先写故事，没故事先写问题
- 别停顿修改，一气写完
- 宁可句子粗糙，别让它光滑

**开头选型**（5选1）：
1. **用事实**："有一年夏天，连续下雨。工地里..."
2. **用对话**："前阵子，一个客户问我..."
3. **用否定**："我不是只讲XX的人。"
4. **用反常**："那个最便宜的选择，最后花得最多。"
5. **用现场**："今天在看的：南宁第X份报价..."

**输出**：原始草稿（必然有AI味，没关系）

---

## ③ AI检测

**做什么**：把草稿扔给 `/dbs-ai-check` 扫描。

**命令**：
```
/dbs-ai-check
```
然后贴草稿。

**输出**：AI指纹报告。标记出：
- 🔴 强信号（"不是X是Y"连发、段段金句、开头三件套）
- ⚠️ 中信号（排比、概念命名、句子节奏均匀）
- 💡 弱信号（翻译腔、内部语言泄露）

**注意**：只做检测，不改。拿到报告后进入下一步。

---

## ④ 赞诺化改写

**做什么**：对照 [赞诺写作DNA](./zeno-writing-dna.md) 和AI指纹报告，逐处修改。

**改写顺序**（按严重度）：
1. 先处理 🔴 强信号（删"不是X是Y"多余项、拆开头三件套、删多余金句）
2. 再处理 ⚠️ 中信号（打破排比、减概念名、变化句子长度）
3. 最后过 💡 弱信号（去翻译腔、把内部语言换成客户端的话）

**每段改完问自己**：
- 这句话我坐在工地上会这么跟客户说吗？
- 有没有超过50字的句子？
- 有没有我嘴上永远不会说的词？

**必须加的**：
- 至少一处真实见闻或经历
- 专业术语的括号解释
- 统一结尾模板

**输出**：定稿（自认为合格）

---

## ⑤ 终审 checklist

**做什么**：用下面的清单逐条过。

```
□ 全文没有"底层逻辑""结构性""赋能""抓手""方法论""闭环""颗粒度"
□ 全文没有"因此""故而""进而""从而""此外"
□ "不是X是Y"不超过1次
□ 每句不超过50字
□ 至少1个真实故事或具体数字（"16年""38份报价""2018年"）
□ 专业术语都有白话解释
□ 结尾用的是统一模板
□ 没有"愿你""你值得""在这个XX的时代"类AI祝福语
□ 读出声来，像人说话不像文章
```

**如果不过**：回到④继续改。**如果过了**：进入⑥。

---

## ⑥ 发布

**做什么**：格式化并发布。

**命令**：
```
/md2wechat
```
把markdown转成微信公众号后台可粘贴的HTML。

**发布前确认**：
- [ ] slug已设置
- [ ] 封面图已准备
- [ ] 分类/tag已标记
- [ ] 结尾链接可用

---

## 附：旧文章批量修改流程

如果需要批量改旧文章（像这次47篇），顺序是：

```
1. 删署名 + 换结尾      → 脚本批量
2. 去书面语 + 拆长句    → 全局替换
3. 术语加白话解释       → 按术语表逐篇加
4. 每篇终审             → checklist逐篇过
5. 补真实案例           → 手动填【真实案例待补】标记
```
